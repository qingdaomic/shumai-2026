> 阶段：树脉 V4.27｜后端变更拆分与风险归档
> 日期：2026-05-08
> 窗口：2号窗口，本地与服务器共同执行任务
> 边界：本阶段只检查、分类、记录；不提交、不推送、不部署、不服务器操作、不重启 PM2、不改数据库、不删除文件。

## 1. 总体结论

V4.26 已把两个干净提交推送到 GitHub，当前剩余工作区不再是 V4 前端静态成果，而是后端、数据库、语音、动画、部署脚本、旧产物、文档和 SKE 前置资料混合状态。

当前不适合直接 `git push`，也不适合服务器整仓 `git pull`。尤其不能把 `server/`、`deploy/`、`public/animations/`、`dist-singlefile/`、TTS / ASR、动画定时任务和数据库 schema 变更混在一个提交或一次部署里。

建议下一步不要直接进入大范围后端上线。更稳的顺序是：

1. 先做后端稳定能力逐文件审查。
2. 再单独处理数据库 / schema 迁移计划。
3. 部署脚本按线上实际 Nginx root `/opt/shumai/dist` 修正后再提交。
4. SKE-1 可以进入设计与种子 Skill 阶段，但代码实施时必须和 TTS / ASR / 动画拆开。

## 3. 分组归档

### A组：后端稳定能力候选

候选文件：

- `server/api/admin.js`
- `server/api/parent.js`
- `server/api/teacher.js`
- `server/api/wechat.js`
- `server/api/svg.js`
- `server/api/ai.js`
- `server/bot.js`
- `server/services/daily.js`
- `server/prompts/tutor.js`

主要内容：

- 管理端资源与套餐统计接口。
- 家长端行动建议、教师端课堂建议。
- 微信绑定上下文、最近互动时间、ClawBot 推送文案升级。
- 每日任务读取学习计划。
- AI 提示词品牌名与讲解风格调整。
- 题目 SVG 生成与缓存能力。

处理建议：

- 可以作为后续后端稳定能力候选，但必须逐接口审查鉴权、输入校验、性能和数据库依赖。
- `server/api/admin.js` 会读取服务器资源状态和 PM2 状态，虽然是管理能力，但涉及系统命令调用，必须先做权限与错误边界审查。
- `server/api/svg.js` 涉及生成缓存表和 AI 调用，应单独检查成本、缓存策略和失败回退。
- `server/bot.js` 与微信私聊上下文强相关，提交前要确认线上 ClawBot token、上下文 token 和主动推送策略不会破坏现有机器人。

### B组：数据库 / schema 变更

候选文件：

- `server/schema.sql`
- 与 schema 强绑定的 API：`server/api/study-plan.js`、`server/api/admin.js`、`server/api/svg.js`、`server/api/tts.js`

当前 schema 变化包括：

- 新增 `study_plans` 表。
- `users` 表新增 `exam_date`、`role`、`wechat_context_token`、`wechat_last_seen`。
- 新增或扩展 `system_config` 的资源包配置。
- 新增 `question_svgs` 表与索引。
- 新增 `tts_usage` 表与索引。

风险：

- 属于数据库影响范围，不能和普通前端或文档提交混在一起。
- 上线前必须先备份数据库，再确认 `ALTER TABLE`、索引和 seed 数据对线上库是幂等、安全的。
- 与 TTS / ASR、SVG 缓存、学习计划有关的表如果部分上线、部分接口未上线，容易形成半成品状态。

处理建议：

- 单独开数据库迁移审查阶段。
- 先写清楚“哪些表已在线上存在、哪些是新增、哪些字段会被现有接口读取”。
- 再决定是继续使用启动时 `initDb()` 自动建表，还是拆出明确迁移脚本。

## 4. 敏感信息风险

本阶段在排除 `local-notes/`、`video/`、`node_modules/` 后做了敏感关键词扫描。

当前没有确认发现明文真实云密钥。扫描结果主要是：

- 环境变量名，例如 `DEEPSEEK_API_KEY`、`TENCENTCLOUD_SECRET_ID`、`TENCENTCLOUD_SECRET_KEY`。
- 文档中的占位示例，例如 `sk-xxx`、`your_key_here`、`your_strong_password_here`。
- 代码里从 `process.env` 读取密钥的正常逻辑。
- ClawBot / JWT / 上下文 token 的变量名或说明。

仍需注意：

- `server/env-template.txt` 只能保留占位值，不能写真实密钥。
- `DEPLOY.md` 和历史部署文档里的示例密码、示例 Key 只能作为占位说明，正式提交前建议统一标注为示例。
- `local-notes/` 继续视为私密目录，不进入报告、不提交、不同步。

## 5. 后续提交建议

建议后续拆成以下任务，而不是一次性提交：

1. 后端稳定能力审查提交：只纳入已审查通过的 admin / parent / teacher / wechat / daily 等能力。
2. 数据库迁移提交：只处理 schema、迁移说明、备份和回滚策略。
3. 部署脚本修正提交：只修正 `/opt/shumai/dist` 等真实线上路径和 dry-run 说明。
4. TTS / ASR 独立阶段：单独处理腾讯云资源包、ASR、TTS 缓存、移动端语音权限和成本统计。
5. 动画 / 视频独立阶段：决定 `public/animations/` 是提交、忽略、外部存储，还是未来改为 B 站链接字段。
6. SKE-1 独立阶段：只做教学 Skill 引擎的数据库结构、种子 Skill 与最小 API，不混入语音、动画和部署。

- 无论选择哪条路，TTS / ASR、动画定时任务、部署脚本和数据库迁移都不要混入 SKE-1。
