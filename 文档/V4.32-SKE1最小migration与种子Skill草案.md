## 3. 为什么 SKE-1 只需要两张表

SKE-1 要解决的不是“把所有学习系统数据都建完”，而是先完成最小闭环：

```text
题目 / 知识点 / 场景 / 学生状态
→ 推荐 3 条“你可以这样问”
→ 学生点击
→ AI 使用对应教学策略
→ 记录展示、点击、调用、反馈
→ 后续按数据优化 Skill
```

这个闭环只需要两类数据：

- Skill 本体：系统有哪些教学策略、适合什么题、什么知识点、什么场景。
- Skill 事件：某个 Skill 是否被展示、点击、调用、反馈有帮助。

因此最小表结构是：

- `prompt_skills`
- `prompt_skill_events`

SKE-1 暂不需要：

- `study_plans`
- TTS / ASR
- SVG 文件缓存
- 微信上下文
- 动画 cron
- 管理端资源统计
- 用户自带模型 Key
- AI 点数计费

## 4. `prompt_skills` 字段设计

`prompt_skills` 保存结构化教学策略。

核心字段：

- `skill_key`：稳定唯一标识，例如 `math_common_hint_first_step_001`
- `name`：后台和调试可读名称
- `type`：Skill 类型，例如 `hint`、`diagnose`、`topic_patch`、`method_hint`
- `audience`：面向对象，第一阶段主要是 `student_frontend`
- `subject`：学科，第一阶段只做 `math`
- `stage`：学段，第一阶段为 `middle`
- `grade`：可选年级
- `topic_code`：适用知识点
- `method_code`：适用方法，例如 `substitution`、`classification`
- `question_type`：适用题型或题库入口
- `scene`：使用场景，例如 `question_detail`、`wrong_fix`、`exam_review`
- `goal`：教学目标
- `difficulty`：难度
- `tone`：语气
- `content`：前台学生可点击的问题文本或 AI 调用时的策略文本
- `model_role`：第一阶段主要是 `user_suggestion`
- `trigger_tags`：匹配标签
- `example_questions`：示例题 ID，例如 `fg01_0`、`tg01_0`
- `weight`：推荐权重
- `version`：版本
- `status`：启用状态

## 5. `prompt_skill_events` 字段设计

`prompt_skill_events` 记录 Skill 是否真的有效。

核心字段：

- `user_id`
- `prompt_skill_id`
- `question_id`
- `question_type`
- `topic_code`
- `method_code`
- `scene`
- `event_type`
- `result`
- `meta`
- `created_at`

建议事件类型：

- `impression`
- `click`
- `ai_used`
- `helpful`
- `not_helpful`
- `correction_done`
- `similar_correct`

后续可用这些事件计算：

- 展示次数
- 点击率
- AI 调用率
- 有帮助率
- 没帮助率
- 某知识点有效 Skill
- 某错因最有效修复方式
- 同类题后续正确率

## 6. 种子 Skill 分类逻辑

本阶段 41 条种子 Skill 聚焦初中数学，不做英语。

覆盖类型：

- 概念修复：例如概念混淆、有理数符号、前置知识链。
- 审题拆解：例如圈关键条件、方程等量关系、统计图表读数。
- 计算纠错：例如逐行检查符号、移项、通分、去括号。
- 方法提示：例如换元法、分类讨论、面积法、数形结合、辅助线。
- 变式训练：例如同知识点变式、基础题到题组。
- 真题迁移：例如十年真题考法、题组迁移到真题。
- 向下补差：例如更简单同类题、三步修复知识点。
- 向上提升：例如基础题提升到题组、题组迁移到真题。
- 压轴分层：例如压轴题三层拆解、动点题变量不变量。
- 错因复盘：例如错题清除标准、方法选错复盘。

覆盖树脉数学根基：

- 基础知识：`topic_patch`、`foundation_patch`
- 基础题：`basic`、`after_basic_mastery`
- 题组训练：`topic_group`
- 压轴题组：`final_group`
- 10 年中考真题：`exam_review`、`exam`
- 23 种方法：`method_code` 覆盖换元法、分类讨论、面积法、参数思想、数形结合、逆向思维、整体思想、辅助线、方程建模等
- 向上提升：`lift`、`transfer`
- 向下补差：`simplify`、`topic_patch`
- 题目、方法、错因、知识点、真题网状关联：通过 `topic_code`、`method_code`、`question_type`、`scene`、`trigger_tags`、`example_questions`、事件表中的 `question_id` 联动

说明：

- 本阶段不是给 1334 道题逐题写 Skill。
- 本阶段先建立可扩展结构，让后续每道题可以按 `question_id`、`topic_code`、`method_code` 和 `scene` 推荐合适 Skill。
- 真正做到“每道题与知识点一一对应”需要后续把现有题库映射关系接入推荐 API。

## 7. 未来推荐逻辑

第一版推荐可以按稳定、可解释的规则做，不急着上复杂模型。

输入：

- `questionId`
- `questionType`
- `topicCode`
- `methodCode`
- `scene`
- `studentState`
- `wrongReason`

推荐顺序：

1. 精确匹配 `topic_code`
2. 精确匹配 `method_code`
3. 匹配 `scene`
4. 匹配 `question_type`
5. 匹配 `status = active`
6. 按 `weight DESC`
7. 不足 3 条时补通用 Skill

学生状态可以影响排序：

- 新手 / 连错：优先 `topic_patch`、`simplify`
- 已做对基础题：优先 `variant`、`lift`
- 真题解析：优先 `transfer`、`exam_review`
- 压轴题：优先 `final_layer`、`method_hint`
- 错题本：优先 `diagnose`、`calculate_fix`、`wrong_review`
