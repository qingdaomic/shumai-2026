# V4.86 服务器 3002 旁路后端演练记录

## 目标

基于 V4.85 已验证通过的干净 release 目录，做一次不抢 3001、不切换线上、不保存 PM2 的 3002 旁路后端演练。

- release 目录中的后端能在生产环境变量下启动
- 临时端口 3002 health 可访问
- 临时 PM2 进程验证后能完整 stop/delete
- 当前线上 3001 服务不受影响

## 执行过程

### 1. 线上基线确认

执行前确认：

- root PM2 中 `shumai-api` 在线
- 当前线上 pid：`700656`
- `http://localhost:3001/api/health` 正常

### 2. 第一次启动尝试

第一次尝试从当前 root PM2 进程复制环境变量，并覆盖：

- `NODE_ENV=production`
- `PORT=3002`

结果：

- 未启动成功
- 原因：当前 PM2 进程环境里存在非标准键名，直接 `export` 时触发 `not a valid identifier`
- 已执行清理
- 3001 health 前后均正常
- root PM2 最终仍只保留 `shumai-api`

### 3. 第二次启动尝试

第二次复制环境变量时增加合法变量名过滤：

- 只导出符合 `[A-Za-z_][A-Za-z0-9_]*=` 的环境变量
- 跳过 `PORT`、`NODE_ENV`、`PWD`、`OLDPWD`、`SHLVL`、`_`
- 再显式设置 `NODE_ENV=production`
- 再显式设置 `PORT=3002`

结果：

- `shumai-api-v4.86` 启动成功
- 临时 pid：`1395067`
- 3002 端口监听成功
- `http://localhost:3002/api/health` 返回 ok
- health 返回内容中 `name` 为 `数脉后端`

### 4. 清理与复核

验证后已执行：

- stop `shumai-api-v4.86`
- delete `shumai-api-v4.86`
- 删除 `/tmp/shumai-api-v4.86.config.cjs`
- 未执行 `pm2 save`

清理后复核：

- root PM2 只剩 `shumai-api`
- 3002 端口不再监听
- `/tmp/shumai-api-v4.86.config.cjs` 不存在
- `http://localhost:3001/api/health` 正常
- release 目录仍存在

## 需要注意的信号

3001 与 3002 health 的 `name` 不一致：

- 3001：`树脉后端`
- 3002：`数脉后端`

这说明 GitHub 最新 release 与当前线上旧代码在后端展示名称上存在差异。正式切换前需要确认该差异是否符合产品命名预期。
