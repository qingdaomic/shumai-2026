# V4.74 SKE/PET 后台操作日志 API 草案

日期：2026-05-09

## 目标

V4.73 已经新增 `admin_operation_logs` migration 草案。V4.74 在不执行 SQL、不部署的前提下，补齐后台操作日志的最小 API 草案，并保证数据库未迁移时安全降级。

## 本阶段完成

- 新增 `server/services/admin-operation-logs.js`
- 在 `server/api/admin.js` 新增两个后台路由：
  - `GET /api/admin/operation-logs`
  - `POST /api/admin/operation-logs`
- 写入日志支持：
  - `skill_weight_adjust`
  - `skill_rewrite`
  - `skill_status_change`
  - `skill_manual_note`
- 目标类型首版只允许 `prompt_skill`
- 数据库表不存在时返回 `stored:false`
- 读取日志默认 50 条，最大 200 条
- 支持按 `action`、`targetType`、`targetId`、`targetKey` 筛选

## 安全边界

- 两个路由继续使用 `authMiddleware + adminGuard`
- 不向前端暴露 token、密钥或环境变量
- `beforeValue` / `afterValue` 只接收对象
- `reason`、`source`、`requestId` 做长度限制
- 不自动写入 Skill 修改接口，避免本阶段扩大行为面

## 降级策略

如果 `admin_operation_logs` 表尚未执行 migration：

- `POST /api/admin/operation-logs` 返回 `ok:true, stored:false`
- `GET /api/admin/operation-logs` 返回空数组和 `stored:false`

这样前端后续可以安全接入：有表时读持久化日志，没表时继续使用 V4.70 的本地临时记录。

## 验证

- `node --check server/services/admin-operation-logs.js`
- `node --check server/api/admin.js`
- `git diff --check`

本阶段没有启动后端服务，没有连接数据库，没有执行 SQL。
