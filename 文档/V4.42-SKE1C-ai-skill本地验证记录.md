## 2. 验证脚本

新增：

- `server/scripts/verify-ai-skill-api.js`

脚本做法：

- 创建本地临时 Express app
- 挂载 `aiRouter` 到 `/api/ai`
- 使用本地测试 JWT
- 强制 `DEEPSEEK_API_KEY=''`，验证 AI Key 缺失时的 fallback
- mock `pool.query`：
  - 学生画像查询返回本地测试数据
  - `prompt_skills` 查询故意抛出“表不可用”，模拟 SKE 表尚未迁移

这样可以在不执行 SQL、不连接线上数据库的前提下，验证 seed Skill fallback 是否工作。

## 3. 验证路径 A：指定 `skill_key`

请求要点：

- `topic_code: quad_fn`
- `question: 二次函数图像经过两个点，怎么先确定解析式？`
- `skill_key: math_topic_quadratic_function_breakthrough_001`
- `mode: hint`

结果：

- HTTP 200
- `ok: true`
- `fallback: true`
- `source: seed`
- `skill.skill_key: math_topic_quadratic_function_breakthrough_001`
- `answer` 非空

说明：

本地 AI Key 被刻意置空，所以返回 fallback 是预期行为。关键点是指定 Skill 能在数据库不可用时从 seed 命中，并进入降级回答。

## 4. 验证路径 B：自动推荐 Skill

请求要点：

- 不传 `skill_key`
- `topic_code: quad_fn`
- `scene: stuck_before_solution`
- `error_type: method_unclear`
- `student_state: stuck`
- `mode: hint`

结果：

- HTTP 200
- `ok: true`
- `fallback: true`
- `source: seed`
- 推荐 Skill：`math_topic_quadratic_function_breakthrough_001`
- `answer` 非空

说明：

SKE 表尚未迁移时，推荐逻辑继续走 seed fallback，并能把二次函数相关 Skill 注入本地降级回答。

## 5. 验证路径 C：缺参 / 异常输入

### `question` 为空

结果：

- HTTP 400
- `error: 缺少问题内容`

说明：

缺少核心问题时返回清晰错误，没有未捕获异常。

### 未知 `topic_code`

请求要点：

- `topic_code: unknown_topic_code`
- `question: 这个题我不知道从哪里开始。`

结果：

- HTTP 200
- `ok: true`
- `fallback: true`
- `source: seed`
- 推荐 Skill：`math_common_hint_first_step_001`
- `answer` 非空

说明：

未知知识点不会导致接口崩溃，会退回通用第一步提示 Skill。
