# V4.51 模型路由与 AI 点数预案记录

最后更新：2026-05-09

## 目标

完成 SKE-6 的最小工程落点：树脉前台不展示模型超市，后端根据教学场景、Skill 类型、题目难度和用户套餐选择合适的 AI 档位，并返回 AI 深度服务额度预览。

## 本阶段完成

- 新增 `server/services/ai-routing.js`
- 建立四个内部档位：
  - `fast`：极速答疑
  - `standard`：标准教练
  - `deep`：深度教练
  - `review`：质检教练
- `/api/ai/skill` 接入 `chooseAiRoute()`
- `/api/ai/skill` 响应新增：
  - `aiRoute`
  - `points`
- `aiRoute.model` 支持环境变量覆盖：
  - `AI_MODEL_FAST`
  - `AI_MODEL_STANDARD`
  - `AI_MODEL_DEEP`
  - `AI_MODEL_REVIEW`

## 点数预案

当前只返回预估点数，不真实扣点，不新增表，不改数据库结构。

预估规则：

- `fast`：1 点
- `standard`：1 点
- `deep`：3 点
- `review`：2 点

后续如果要真实扣点，可以在用户订阅或专门的 AI usage 表中落账。

## 路由原则

- 简单提示、第一步、读题眼：走 `fast`
- 日常讲题、普通错因：走 `standard`
- 压轴题、综合题、深度诊断：VIP / Pro 可走 `deep`
- 免费用户遇到深度信号时先降到 `standard`
- 质检类场景走 `review`

## 边界

- 不部署
- 不 SSH
- 不 PM2 / Nginx
- 不执行 SQL
- 不改 `server/schema.sql`
- 不真实扣点
- 不把模型选择暴露到学生端
