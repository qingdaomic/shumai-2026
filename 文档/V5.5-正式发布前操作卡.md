# V5.5 正式发布前操作卡

日期：2026-05-09

## 目标

把下一次正式发布拆成可执行、可验证、可回滚的操作卡。

本阶段只写操作卡，不连接服务器，不部署，不同步 `/opt/shumai/dist`，不切换 PM2，不 reload Nginx，不执行 SQL。

如果下一阶段进入真实服务器发布、PM2 切换、Nginx 操作或数据库操作，应切换到 GPT-5.5 high，并在执行前再次取得用户明确授权。

## 发布范围选择

建议下一次正式发布先只做前端静态发布。

原因：

- V5.3 build 产物与当前公网 asset 同名：`/assets/index-BBjZDaHT.js`
- 后端正式服务已经稳定在 `shumai-api-v4.94`
- 当前没有必须切换 3001 的新后端代码需求
- 先发布前端 root 可验证部署脚本路径和备份回滚流程

后端切换应作为独立阶段，除非发布前发现本次 release 的后端必须上线。

## 发布前只读核验

执行前先只读确认：

```bash
date
curl -fsS https://shumai.cc/api/health
sudo pm2 status --no-color
sudo nginx -T 2>/dev/null | grep -n "root .*shumai"
grep -o '/assets/index-[^"]*\.js' /opt/shumai/dist/index.html | head -5
cd /opt/shumai-releases/v5.3-20260509-204146/repo
git rev-parse HEAD
git status --short | wc -l
test -f dist/index.html
du -sh dist
```

必须满足：

- health 正常
- PM2 只有 `shumai-api-v4.94:online`
- Nginx root 仍为 `/opt/shumai/dist`
- V5.3 release 工作区脏改数为 `0`
- V5.3 `dist/index.html` 存在

## 备份

正式发布前创建一次时间戳备份：

```bash
STAMP=$(date +%Y%m%d-%H%M%S)
SNAP=/home/ubuntu/shumai-backups/v5.6-frontend-release-$STAMP
mkdir -p "$SNAP"
sudo rsync -a /opt/shumai/dist/ "$SNAP/opt-shumai-dist-before/"
sudo pm2 status --no-color > "$SNAP/pm2-before.txt"
curl -fsS https://shumai.cc/api/health > "$SNAP/public-health-before.json"
grep -o '/assets/index-[^"]*\.js' /opt/shumai/dist/index.html | head -5 > "$SNAP/frontend-asset-before.txt"
```

备份完成后确认：

```bash
test -f "$SNAP/opt-shumai-dist-before/index.html"
test -s "$SNAP/public-health-before.json"
cat "$SNAP/frontend-asset-before.txt"
```

## 前端发布步骤

仅同步 V5.3 release 的前端 `dist/`：

```bash
sudo rsync -a --delete /opt/shumai-releases/v5.3-20260509-204146/repo/dist/ /opt/shumai/dist/
```

说明：

- 不 reload Nginx
- 不改 Nginx 配置
- 不重启 PM2
- 不执行 SQL
- 不修改 `/opt/shumai` 历史目录

## 发布后验证

发布后立即验证：

```bash
curl -fsS https://shumai.cc/api/health
curl -fsS https://shumai.cc/ | head -5
grep -o '/assets/index-[^"]*\.js' /opt/shumai/dist/index.html | head -5
sudo pm2 status --no-color
```

页面级验证建议：

- `https://shumai.cc/`
- `https://shumai.cc/?view=admin`
- `https://shumai.cc/api/health`

验收要点：

- 公网 health 正常
- 首页 HTTP 200
- 后台入口 HTTP 200
- PM2 仍只有 `shumai-api-v4.94:online`
- 前端 asset 符合预期

## 回滚

如果发布后前端异常，立即回滚静态文件：

```bash
sudo rsync -a --delete "$SNAP/opt-shumai-dist-before/" /opt/shumai/dist/
curl -fsS https://shumai.cc/api/health
grep -o '/assets/index-[^"]*\.js' /opt/shumai/dist/index.html | head -5
```

如果后端异常：

- 本阶段不应出现后端异常，因为不切换 PM2
- 若 health 异常，先只读查看 `pm2 status` 和日志，再停止发布流程

## 停止条件

出现任一情况立即停止，不继续发布：

- `nginx -T` 显示 root 不是 `/opt/shumai/dist`
- V5.3 release 工作区不干净
- V5.3 `dist/index.html` 不存在
- 发布前 health 不正常
- 发布前 PM2 不只包含 `shumai-api-v4.94:online`
- 备份目录创建失败
- 备份后无法找到 `index.html`
- 用户未明确授权真实发布
