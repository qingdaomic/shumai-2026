# V4.41 SKE-1C `/api/ai/skill` 最小实现记录

> 阶段：树脉 V4.41｜SKE-1C `/api/ai/skill` 设计与最小实现  
> 日期：2026-05-08  
> 窗口：2号窗口，本地与服务器共同执行任务  
> 边界：本阶段只做本地后端最小实现与记录；不 push、不部署、不服务器操作、不重启 PM2、不执行 SQL、不修改数据库。

## 1. 本阶段目标

V4.33 已实现 SKE 推荐 API 与事件记录：

- `server/services/prompt-skills.js`
- `server/api/skills.js`

V4.39 已把 `/api/skills` 干净挂载到 `server/index.js`。

V4.41 的目标是让 AI 回答真正能使用推荐 Skill：

- 新增 `POST /api/ai/skill`
- 如果传入 `skill_key`，优先使用指定 Skill
- 如果不传 `skill_key`，调用推荐逻辑取 1 条最合适 Skill
- 把 Skill 内容注入现有树脉学长提示词风格
- AI 不可用或未配置 Key 时返回清晰 fallback，不让接口崩死

## 2. 新增服务层

新增：

- `server/services/ai-skill.js`

职责：

- 标准化 `/api/ai/skill` 请求参数
- 按 `skill_key` 查找数据库 Skill，数据库不可用时查找 seed Skill
- 未指定 `skill_key` 时复用 `recommendPromptSkills()` 推荐 1 条 Skill
- 组合树脉学长学生画像、题目信息和 Skill 内容，生成系统提示词
- 提供本地降级回答 `buildAiSkillFallback()`

本服务不执行 SQL migration，不修改 `server/schema.sql`。

## 3. API 设计

新增路由：

```text
POST /api/ai/skill
```

请求体支持：

```json
{
  "question": "这道二次函数题第一步看哪里？",
  "question_id": "fg01_0",
  "topic_code": "quad_fn",
  "stage": "middle",
  "error_type": "method_choice",
  "student_state": "stuck",
  "scene": "question_detail",
  "skill_key": "math_topic_quadratic_function_breakthrough_001",
  "mode": "hint"
}
```

返回：

```json
{
  "ok": true,
  "answer": "树脉学长回答内容",
  "skill": {
    "skill_key": "math_topic_quadratic_function_breakthrough_001",
    "name": "二次函数破局"
  },
  "source": "db 或 seed",
  "reason": "Skill 选择原因",
  "fallback": false
}
```

如果 AI Key 未配置或 AI 服务不可用：

```json
{
  "ok": true,
  "answer": "本地降级提示",
  "skill": null,
  "source": "fallback",
  "reason": "AI 服务暂时不可用，已返回本地降级提示",
  "fallback": true
}
```

## 4. 实现原则

- 复用 `server/api/ai.js` 的 DeepSeek 调用方式。
- 复用 `server/prompts/tutor.js` 的 `buildStudentContext()`。
- 复用 `server/services/prompt-skills.js` 的推荐能力。
- 首版只做文字回答。
- 不接 TTS / ASR。
- 不接动画。
- 不接图片。
- 不修改数据库。
- 不要求 SKE 表已迁移，seed fallback 可继续工作。
