## 1. 本阶段目标

V4.44 已经确定提问斜杠的产品形态：

- 输入 `/` 触发
- 下拉式菜单
- 展示 3-5 条推荐提问词
- 点击提问词后直接发送
- 推荐来自 `/api/skills/recommend`
- 回答调用 `/api/ai/skill`
- 事件调用 `/api/skills/event`
- 前端落点优先 `AskTutor`

V4.45 的目标是先把这条链路在题目教练场里跑起来，不重建题目页，不接树脉搜索，不做复杂个性化。

## 2. 定位结果

本阶段定位到：

- `AskTutor` 是题目页、真题刷题页、错题修复页复用的“问学长”组件。
- AI 输入框状态是 `followUp`。
- 原有 AI 发送函数是 `doAsk()` / `handleFollowUp()`，底层调用 `askTutorAI()`。
- 题目上下文来自 `q`，通过 `normalizeTutorQuestion(q)` 可拿到 `id`、`topic`、`type`、`methods`、`content` 等信息。
- `mode` 可辅助判断场景：`hint` 对应 `stuck_before_solution`，`wrong` 对应 `wrong_fix`，其它默认 `question_detail`。

## 3. 实现内容

主要修改：

- `src/App.jsx`

新增能力：

- 新增 `SLASH_FALLBACK_PROMPTS`，包含 5 条首版 fallback 提问词。
- 新增 `SlashPromptMenu` 小组件，只负责下拉式展示和选择。
- `AskTutor` 监听追问输入框内容，输入 `/` 或末尾 `/` 时打开菜单。
- 打开菜单时优先请求 `POST /api/skills/recommend`，请求失败或未登录时使用本地 fallback。
- 推荐不足 5 条时用本地 fallback 补齐。
- 点击提问词后关闭菜单、清空 `/`，直接调用 `POST /api/ai/skill`。
- 点击前后不会阻断原有问学长按钮、语音追问、普通追问流程。
- `POST /api/skills/event` 已接入 `impression` 和 `click`，失败时静默忽略。

首版 fallback 提问词：

- 先提醒我第一步
- 帮我找题眼
- 我卡在哪个知识点
- 给我一个不直接泄答案的提示
- 换一道同类题练一下

## 4. UI 处理

桌面端：

- 菜单贴输入框下方。
- 宽度跟随输入框。
- 下拉内部纵向滚动，不出现横向滚动。

手机端：

- 菜单贴输入框上方，减少被键盘遮挡的概率。
- 追问输入框在窄屏下独占一行，语音和发送按钮下移，避免菜单被挤成竖排。
- 触摸项高度不低于 44px。
- 文本自然换行，不横向滚动。

主题：

- 使用现有 `C.s1`、`C.border`、`C.text`、`C.geo` 等主题变量，适配深色 / 浅色 / 护眼 / 暖棕。
