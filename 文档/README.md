# 数脉 ShuMai 中考数学智能学习系统

## 项目概述

`shumai-v7-1.jsx` 是一个单文件 React 应用，面向青岛中考数学复习场景，整合了知识图谱、分模块学习、真题训练、错题本、智能诊断、命题预测和晨读卡等能力。

该系统的核心目标是：

- 以知识点为主线组织学习内容
- 以中考真题数据驱动复习优先级
- 以知识依赖关系追溯薄弱根因
- 以错题和掌握状态形成个性化学习路径
- 以 AI 辅助生成诊断、预测和语音讲解内容

当前项目已经额外生成了一个可直接双击打开的离线单文件版本：

- `index.html`

该 `index.html` 是由 `shumai-v7-1.jsx` 一次性编译得到，运行时不依赖 Node / npm / React 开发环境。

## 核心特性

- **知识图谱学习**
  - 以 24 个核心知识节点为图谱主干
  - 支持节点点击跳转到知识详情
  - 支持已掌握状态可视化

- **四模块学习法**
  - 基础知识
  - 基础习题
  - 题组训练
  - 压轴题组

- **23 种解题方法体系**
  - 将方法与知识点建立映射
  - 支持方法分类浏览与关联知识点跳转

- **中考真题训练**
  - 内置多年真题数据集 `EXAM_QS`
  - 支持按知识点、年份、城市、难度筛选
  - 支持错题收录与错题本联动

- **错题本**
  - 自动按知识点分组
  - 展示错因追溯和补差路径
  - 支持语音讲解文稿生成与朗读

- **智能诊断**
  - 分析未掌握高频考点
  - 统计错题分布
  - 沿知识依赖树追溯根源
  - 自动生成推荐学习路径
  - 调用 AI 输出学情诊断报告

- **中考预测**
  - 基于近年真题统计生成命题预测
  - 输出必考点、高概率点、复习优先级与方法建议

- **每日晨读卡**
  - 基于 `MORNING_DATA` 构造知识点晨读内容
  - 支持卡片式记忆和结果记录

- **本地持久化**
  - 使用 `localStorage` 保存掌握状态、错题、基础错题、AI 设置等

## 页面结构

应用的页面由 `view` 状态驱动，导航配置位于 `NAV`。

### 页面列表

- **首页 `home`**
  - 入口仪表盘
  - 展示掌握进度、知识域统计、错题本统计、快捷入口等

- **晨读卡 `morning`**
  - 每日记忆与快速回顾页面

- **知识图谱 `graph`**
  - 可视化展示知识点节点与关联关系

- **四模块 `modules`**
  - 系统化分层学习主页面

- **知识详情 `detail`**
  - 单个知识点详情页
  - 展示讲解、前置知识、典型题与练习入口

- **方法体系 `methods`**
  - 23 种解题方法分类浏览

- **真题训练 `practice`**
  - 真题库浏览、筛选、解析和错题收录

- **错题本 `wrong`**
  - 按知识点聚合错题
  - 展示根因分析和语音讲解

- **智能诊断 `diag`**
  - 学情诊断、风险预警、学习路径与 AI 诊断报告

- **中考预测 `predict`**
  - 基于历年数据的命题趋势预测

## 数据结构概览

### 1. 主题色设计系统 `C`

定义了系统统一色板，包括：

- 背景色
- 文字色
- 边框色
- 代数 / 几何 / 统计等功能色
- 四模块主题色
- 方法体系色

### 2. 解题方法 `METHODS`

每个方法对象通常包含：

- `id`
- `name`
- `cat`
- `desc`
- `topics`
- `example`

用于方法体系页面与知识点联动。

### 3. 知识点 `TOPICS`

知识图谱核心数据，定义了中考数学的重点知识点集合。

常见字段包括：

- `id`
- `name`
- `domain`
- `freq`
- `examYears`
- `totalScore`
- 坐标 / 关联 / 前后置等扩展信息

### 4. 真题题库 `EXAM_QS`

系统题库主数据源，字段通常包括：

- `id`
- `yr`
- `no`
- `city`
- `type`
- `topic`
- `score`
- `diff`
- `subTopics`
- `methods`
- `content`
- `answer`
- `sol`
- `error`
- `upLink`
- `downLink`

### 5. 晨读数据 `MORNING_DATA`

为每个知识点提供：

- `slogan`
- `trap`

用于晨读卡生成。

### 6. 模型配置 `AI_MODELS`

当前支持：

- DeepSeek 系列
- 豆包系列

并通过 `window.__SHUMAI_MODEL__`、`window.__SHUMAI_DSKEY__`、`window.__SHUMAI_DBKEY__` 与工具函数联动。

## 核心逻辑说明

### 路由与导航

应用没有使用外部路由库，而是通过根组件里的状态切换页面：

- `view`
- `detailId`

入口函数：

- `navigate(v, tid = null)`

逻辑：

- 如果传入 `tid`，进入 `detail` 页并记录知识点 ID
- 否则切换普通页面视图

此外还监听了全局事件：

- `shumai-nav`

用于子组件触发跨页面跳转。

### 本地状态持久化

根组件通过 `localStorage` 管理以下状态：

- `mastered`
- `wrongSet`
- `basicWrongSet`
- `aiModel`
- `dsKey`
- `dbKey`

存储键：

- `shumai_v7`

保存方式：

- 初次加载时从本地恢复
- 状态变化时自动写回

### 知识根因追溯

核心函数：

- `traceRootCause(topicId, masteredSet)`

逻辑：

- 基于 `DEPENDENCIES` 进行 BFS 向下搜索
- 找出当前知识点依赖链中未掌握的前置知识
- 返回：
  - `unmastered`
  - `roots`

该函数被以下模块复用：

- 错题本
- 智能诊断
- 学习路径推荐

### AI 调用

兼容调用入口：

- `callAI(system, user, history = [])`
- `callClaude = callAI`

能力包括：

- 智能诊断报告生成
- 中考预测报告生成
- 错题语音讲解文稿生成

注意：

- 需要联网
- 需要配置有效 API Key
- 未配置时会返回提示文本，不会自动完成云端分析

### 语音讲解

相关函数：

- `fetchMSTTS(text)`
- `speakText(text, onStart, onEnd)`
- `fallbackSpeak(text, onStart, onEnd)`
- `stopSpeak()`
- `genVoiceScript(q, topicName)`

机制：

- 优先使用微软 Edge 朗读接口合成音频
- 失败时回退到浏览器原生 `speechSynthesis`

## 已知实现特点与限制

- **单文件实现规模大**
  - 所有设计、数据、逻辑、页面都在一个 `jsx` 文件中
  - 便于单文件交付，但不利于长期维护

- **内置数据量大**
  - 真题、知识点、方法体系、晨读数据都写在源码里
  - 易于离线分发，但会使文件体积变大

- **AI 能力依赖网络**
  - 离线单文件版 UI 可运行
  - 但 AI 生成能力仍依赖外部 API

- **存在少量数据重复键警告**
  - 某些题目对象里有重复的 `answer` 或 `sol` 字段
  - 构建可通过，但后者会覆盖前者
  - 建议后续做数据清洗

## 建议的后续优化

- **拆分源码结构**
  - 将颜色系统、数据集、工具函数、页面组件拆成多个文件

- **建立正式构建流程**
  - 增加 `package.json`
  - 引入标准打包与开发脚本

- **修复数据质量问题**
  - 清理题库对象重复字段
  - 为数据建立校验脚本

- **完善错误边界**
  - 给各页面增加错误边界组件，避免局部异常导致整页黑屏

- **增强离线能力**
  - 为 AI 功能提供离线占位说明或本地规则版本

---

## 维护建议

如果继续以单文件维护 `shumai-v7-1.jsx`，建议遵循以下原则：

- 按区域分块维护
- 保持数据结构一致性
- 所有新增页面统一走 `NAV + view` 机制
- 所有新增持久化状态统一收口到 `loadStorage / saveStorage`
- 所有 AI 新能力统一复用 `callAI`
