## 代码依赖分析

### 学生端 / AI Skill 推荐

相关文件：

- `server/services/prompt-skills.js`
- `server/services/ai-skill.js`

结论：

- 推荐逻辑优先读 `prompt_skills`
- 数据库不可用或表未迁移时，会降级读取 `server/seeds/prompt_skills_seed.json`
- 事件记录写入 `prompt_skill_events`
- 表不存在时事件记录降级为 no-op

影响：

- 学生端推荐不会因为缺表直接中断
- 但事件不会持久化，质量统计为空

### 后台 Skill 管理

相关文件：

- `server/api/admin.js`

结论：

- `GET /api/admin/skills` 直接查询 `prompt_skills` 与 `prompt_skill_events`
- `PUT /api/admin/skills/:id` 直接更新 `prompt_skills`
- 当前没有 seed fallback
- 当前缺表时后台 Skill 管理会返回 500

影响：

- 如果正式切换 release 后端，但不补 SKE 表，后台 Skill 管理不可用
- 后台操作日志页面本身已经有 `admin_operation_logs`，但操作日志关联的 Skill 操作无法完整验证

## 与后台操作日志上线的关系

V4.88 已经创建 `admin_operation_logs`，因此操作日志表本身可用。

但后台操作日志的真实业务验证需要触发：

- Skill 降权
- Skill 改写
- Skill 操作历史查看

这些动作依赖 `prompt_skills`。所以正式上线验证前至少需要：

- 创建 `prompt_skills`
- 创建 `prompt_skill_events`
- 导入 41 条种子 Skill

否则只能验证 `admin_operation_logs` 的底层读写，不能完整验证后台 Skill 操作链路。

## 推荐执行顺序

建议下一阶段不要直接切换 release 后端，而是先执行 SKE 最小 schema：

1. 基于 V4.87 数据库备份和 V4.88 migration 记录，确认当前数据库状态
2. 执行 `server/migrations/20260508_ske_minimal.sql`
3. 验证两张表存在
4. 验证 7 个索引存在
5. 导入 `server/seeds/prompt_skills_seed.json`
6. 验证 `prompt_skills` 记录数为 41
7. 验证 `skill_key` 唯一
8. 启动 3002 旁路后端
9. 验证 `GET /api/admin/skills` 不再因缺表失败
10. 再验证后台操作日志 API 写读

## 风险与边界

SKE 最小 migration 是 additive，但仍需注意：

- `prompt_skill_events.user_id` 外键依赖 `users(id)`
- `prompt_skill_events.prompt_skill_id` 外键依赖 `prompt_skills(id)`
- 种子导入必须幂等，建议 `ON CONFLICT (skill_key) DO UPDATE` 或 `DO NOTHING`
- 如果只建表不导种子，后台列表为空，无法验证 Skill 操作
- 如果直接导种子但字段不匹配，会造成上线前卡点
