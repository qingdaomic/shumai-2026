## 真实上线验证前置授权

以下动作任何一个都必须等用户明确授权后才能执行：

- SSH 登录服务器
- 在服务器执行 `git pull`
- 执行 PostgreSQL 备份
- 执行 migration SQL
- 重启 PM2 后端进程
- 修改 Nginx / PM2 / 服务器环境变量
- 访问或改动线上数据库
- 线上部署前端或后端

可接受的授权表达示例：

> 授权连接服务器并执行 V4.82 后台操作日志上线验证。

## 授权后执行清单

### 1. Git 与版本确认

- 本地确认 `origin/main` 已包含 V4.73-V4.80 相关 commit
- 服务器确认 `/opt/shumai` 当前分支和状态
- 服务器确认最近 commit 是否已经拉到最新

建议命令只在授权后执行：

```bash
cd /opt/shumai
git status -sb
git log --oneline -10
```

### 2. 数据库备份

执行 migration 前必须先备份 PostgreSQL。

建议边界：

- 优先做完整备份
- 记录备份文件路径、时间、数据库名
- 备份成功后再执行 migration
- 如果备份失败，停止上线验证

### 3. Migration 执行

目标文件：

- `server/migrations/20260509_admin_operation_logs.sql`

执行边界：

- 只执行这一份 migration
- 不修改 `server/schema.sql`
- 不手写临时 SQL
- 不执行 drop / truncate
- 如果表已存在，确认 migration 的幂等行为后再继续

### 4. 后端部署与重启

后端变更已经在 V4.74 接入 `server/index.js`。授权后可以按当前项目部署规范执行：

- 服务器拉取最新 `main`
- 如 `package.json` 未变化，不主动安装依赖
- 重启 `shumai-api`
- 检查 PM2 状态与后端健康检查

### 5. API 验证

上线后重点验证：

- 未登录访问后台日志接口应被拒绝
- 非管理员访问后台日志接口应被拒绝
- 管理员读取日志接口可用
- 数据库表不存在时的降级逻辑不再触发
- 写入接口能保存 `operation_type`、`target_type`、`target_id`、`before_snapshot`、`after_snapshot`、`reason`
- 读取接口能按 `target_type` / `target_id` 返回最近记录

### 6. 前端验证

在后台 Skill 管理中验证：

- 人工降权成功后，最近操作写入线上日志
- 人工改写成功后，最近操作写入线上日志
- Skill 行内最近操作提示可见
- 操作历史入口可打开
- 历史详情中文摘要可展开
- 全部 / 降权 / 改写筛选可切换
- 页面不再只显示本地降级记录

### 7. 回滚边界

本次日志表属于新增审计能力，理论上不影响学生端主链路。若上线验证失败：

- 优先停止继续操作，保留现场日志
- 如果是后端接口错误，可回退代码版本并重启 PM2，但必须再次获得确认
- 如果是 migration 失败，不直接 drop 表，不直接删数据
- 如果是前端显示问题，可先让前端继续使用本地降级记录
- 所有数据库破坏性回滚动作必须单独授权
