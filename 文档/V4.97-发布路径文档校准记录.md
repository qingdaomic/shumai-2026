## 当前发布路径事实

V4.96 已通过公网真实页面验收确认：

- 当前 Nginx 实际前端 root：`/opt/shumai/dist`
- 当前公网首页加载的新资源：`/assets/index-BBjZDaHT.js`
- 当前后端 PM2：`shumai-api-v4.94:online`
- 当前后端 health：`ok:树脉后端`

因此，后续前端静态发布默认目标应为：

```bash
/opt/shumai/dist
```

而不是：

```bash
/var/www/shumai
```

## `/var/www/shumai` 的处理原则

`/var/www/shumai` 在多份历史记录中出现，原因是早期部署文档和 V4.94 操作卡曾把它当作前端发布目录。

V4.96 之后的原则：

- 不修改历史执行记录中的事实描述
- 不把 `/var/www/shumai` 作为当前默认发布目标
- 只有在重新执行 `nginx -T` 并确认 Nginx root 已变更时，才允许更新发布目标
- 若继续备份 `/var/www/shumai`，应明确它是历史/旁路目录，而不是当前公网 root

## 后续前端发布最小准则

正式发布前：

1. 只读确认 Nginx 当前 root。
2. 记录当前公网首页引用的 asset。
3. 备份 Nginx 实际 root。
4. 从已验证 release 的 `dist/` 同步到 Nginx 实际 root。
5. 重新验证公网首页 asset、后台入口和 `/api/health`。

当前建议命令语义：

```bash
# 只读确认，实际执行前仍需人工核对输出
sudo nginx -T | grep -n "root .*shumai"

# 当前线上事实对应的发布目标
sudo rsync -a --delete <release>/repo/dist/ /opt/shumai/dist/
```

注意：上面命令是操作准则记录，本阶段没有执行。
