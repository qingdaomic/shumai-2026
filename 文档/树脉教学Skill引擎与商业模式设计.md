# 树脉教学 Skill 引擎与商业模式设计

> 本文档用于沉淀树脉未来战略：提示词学习引擎、教学 Skill 系统、AI 模型路由、免费共建商业模式。  
> 最后更新：2026-05-08  
> 状态：战略设计稿，供后续实施窗口拆任务落地。

---

## 一、战略总纲

树脉不是传统题库，也不是简单 AI 答题工具。树脉要建设的是一个会随着学生使用而持续变聪明的学习系统。

核心判断：

- 题库解决“学什么”
- AI 解决“怎么讲”
- 教学 Skill 引擎解决“什么时候、对谁、用哪种方式讲最有效”
- 学生数据不是单纯增长，而要转化为错因网络、修复网络、路径网络和提示智能

树脉未来真正的护城河不是题目数量，而是：

**真实学生学习过程里，被不断验证、筛选、优化出来的教学策略库。**

## 二、免费、共建与数据生长

### 2.1 免费的意义

树脉基础学习能力应尽可能长期免费。免费不是低价策略，而是让系统获得足够真实的学习场景和学生行为数据。

免费带来的价值链：

```text
更多学生使用
→ 产生真实学习轨迹
→ 系统识别错因、卡点、有效提示和修复路径
→ AI 教练越来越懂学生
→ 后来的学生学得更快、更清楚
→ 口碑增长
→ 更多学生进入
```

这不是普通流量飞轮，而是学习智慧飞轮。

### 2.2 数据生长与智力生长的区别

数据生长只是记录：

- 哪些题被做过
- 哪些题错得多
- 哪些知识点薄弱
- 哪些 AI 讲解被点击
- 哪些页面访问量更高

智力生长是系统真正变聪明：

- 知道某类学生为什么会错
- 知道一道题最适合先提示哪一步
- 知道什么时候该给完整解析，什么时候只给突破口
- 知道某个错因要补哪个前置知识点
- 知道哪句提示能让学生继续思考
- 知道哪种讲法对七年级学生听得懂，对九年级学生太啰嗦

树脉要把数据生长变成智力生长。

### 2.3 成本公开与共建模式

建议采用：

**基础学习长期免费 + 成本透明共建 + 高成本服务按需付费。**

可公开的成本包括：

- 服务器成本
- 域名和 HTTPS 等基础成本
- AI 模型调用成本
- 短信验证码成本
- 图片 OCR / TTS / 存储成本
- 运维和安全成本
- 后续教研与服务人员成本

当用户规模超过 1000 人、3000 人、10000 人等节点时，可以公开月度成本，并提供“树脉共建计划”。

共建计划不是“买特权”，而是共同维持一个基础学习系统继续免费。

## 三、教学 Skill 引擎的定义

### 3.1 它是不是 Skill

可以理解为教育场景里的 Skill 系统，但树脉的教学 Skill 引擎比普通 skill 更细、更动态。

普通 skill 通常是：

- 一组提示词
- 一个固定任务流程
- 一种角色设定
- 一个工具调用能力

树脉教学 Skill 引擎是：

```text
题目 + 知识点 + 学生状态 + 错因 + 场景 + 模型能力
→ 选择合适的教学 Skill
→ 组合后台专家提示词和前台引导提示词
→ 调用 AI 模型
→ 记录学习效果
→ 调整 Skill 权重和版本
```

因此它是：

**Skill Library + Prompt Repository + Teaching Workflow + Feedback Optimizer。**

### 3.2 一句话定义

**树脉教学 Skill 引擎：把题目、知识点、学生状态、错因和模型能力组合起来，动态选择最有效的教学策略。**

## 四、提示词仓库的两类作用

### 4.1 后台专家提示词

后台专家提示词用于调用 DeepSeek、OpenAI、Claude、Gemini 等通用大模型，让模型输出更像树脉学长、数学老师、学习诊断师，而不是普通聊天机器人。

典型类型：

- 单题讲解提示词
- 错因诊断提示词
- 知识点总结提示词
- 只给一步提示的提示词
- 追问式引导提示词
- 变式题生成提示词
- 纸质训练单生成提示词
- 家长沟通建议提示词
- 教师课堂建议提示词
- 答案质量检查提示词

这类提示词决定 AI 输出的专业度。

### 4.2 前台学生引导提示词

前台提示词展示给学生，帮助学生知道自己可以怎么问，也帮助学生形成更好的提问方式和思考方式。

例如题目详情页可以展示：

- 给我一个突破口
- 先别说答案，提示第一步
- 帮我找题干里的关键条件
- 判断我错在哪里
- 给我一道更简单的同类题
- 这题和哪个知识点有关
- 如果我看不懂题干，应该先圈出哪些信息

前台提示词不是装饰，而是学习行为引导。

它的隐藏价值：

- 降低学生提问门槛
- 减少直接要答案
- 训练学生提问能力
- 让学生学会从“答案导向”转向“理解导向”
- 为系统收集哪种提示有效的数据

## 五、Skill 类型体系

第一版建议先定义以下 Skill 类型：

| 类型 | 作用 | 示例 |
|------|------|------|
| explain | 完整讲解 | 给出 ①②③ 步骤、核心知识点、易错点 |
| hint | 只给提示 | 只提示第一步，不直接给答案 |
| socratic | 追问引导 | 通过一个问题推动学生自己想 |
| diagnose | 错因诊断 | 判断是概念错、审题错、计算错还是方法错 |
| simplify | 降阶讲解 | 生成更简单的同类题或更基础的解释 |
| variant | 变式生成 | 生成同知识点、同方法或降难度题 |
| topic_patch | 知识补洞 | 补前置知识点，解释基础概念 |
| plan | 学习规划 | 生成今日任务、周计划、考前冲刺节奏 |
| parent | 家长建议 | 给家长今晚怎么说、不要怎么催 |
| teacher | 教师建议 | 给教师明天课堂讲评重点 |
| quality_check | 质量检查 | 检查 AI 回答是否正确、是否适合初中生 |

## 六、结构化提示词资产

提示词不能只是文本文件，应成为结构化资产。

建议字段：

```text
id: skill_hint_quad_fn_001
name: 二次函数突破口提示
type: hint
audience: student_frontend
subject: math
stage: middle
grade: 9
topic_code: quad_fn
question_type: function
scene: stuck_before_solution
goal: give_first_breakthrough
difficulty: medium
tone: calm_coach
content: 先看题目要你求什么，再找图像里能确定的点或对称轴。二次函数题的第一步通常不是算，而是定结构。
model_role: user_suggestion
weight: 0.70
version: 1.0.0
status: active
created_at: 2026-05-08
updated_at: 2026-05-08
```

效果字段可以由统计表计算，不一定全部存在主表：

- 展示次数
- 点击次数
- AI 调用次数
- 有帮助率
- 负反馈率
- 点击后继续作答率
- 点击后完成订正率
- 同类题后续正确率
- 平均对话轮数
- 平均消耗 token

## 七、前台产品形态

### 7.1 题目教练场：你可以这样问

题目详情页、真题解析区、错题修复页都可以出现轻量气泡：

```text
你可以这样问我

[给我一个突破口]
[先别说答案，提示第一步]
[帮我找关键条件]
[判断我错在哪里]
[给我一道更简单的同类题]
```

学生点击后，系统把对应 prompt skill 带入 AI 对话。

### 7.2 AI 回答分层

不要所有按钮都给完整答案。AI 响应应有层级：

- 一步提示
- 思路引导
- 追问
- 错因诊断
- 完整讲解
- 同类训练
- 知识补洞

这会让树脉从“答案机器”变成“私人教练”。

### 7.3 前人踩坑

当数据足够后，每个知识点和题型可以展示系统精选的“前人踩坑”：

```text
很多同学第一次会把这里理解错：
不是先展开计算，而是先确定等量关系。

最常见错因：
把“相等的量”看成“相似的量”。

修复方式：
先做 3 道等量关系识别题，再回到这道题。
```

这不是普通社区，而是结构化学习智慧。

### 7.4 提问斜杠：学生卡住时的思维扶手

提问斜杠是 SKE 在学生端最重要的外显入口之一。学生在题目教练场输入框里输入 `/`，系统不弹普通快捷菜单，而是根据当前题目、知识点、错因、学生基础和历史点击效果，推荐 3-5 条最适合此刻的提问词。

它的作用不是让学生更快要答案，而是让学生更快问出能开窍的问题：

- 先提醒我第一步
- 帮我找题眼
- 我卡在哪个知识点
- 给我一个不直接泄答案的提示
- 换一道同类题练一下

如果系统判断学生常错在审题，优先推荐：

- 帮我圈出关键信息
- 这题条件怎么翻译成式子
- 我是不是漏看了隐藏条件

如果系统判断学生常错在方法选择，优先推荐：

- 这题该用哪种方法
- 有没有更快解法
- 这题和哪类真题相似

第一版交互建议采用下拉式：桌面端贴着输入框下方出现，手机端贴着输入框上方出现。暂不采用卷帘式或瀑布式，避免打断做题节奏。

内部能力名可称为 `Slash Prompt Coach`，学生端中文名建议为“提问斜杠”。它和 SKE 的关系是：

- SKE 决定推荐哪几条教学 Skill
- 提问斜杠负责把推荐变成学生可点击、可理解的问法
- 展示记录为 `impression`
- 点击记录为 `click`
- 后续 AI 调用记录为 `ai_used`
- 有帮助反馈记录为 `helpful`

长期看，提问斜杠会让树脉知道：哪个学生、在哪类题、哪种卡点下，哪一句提示最容易让他继续思考并做对题。

### 7.5 树脉搜索：从找资料升级为找补法

树脉需要搜索，但不应做成普通资料站搜索。搜索栏应成为学生和家长进入系统的第二入口：

- 第一入口：今日学习驾驶舱，系统告诉学生今天最该做什么。
- 第二入口：树脉搜索，学生主动寻找知识点、题目、方法、错因和真题。

树脉搜索应支持：

- 知识点：一次函数、二次函数、圆、相似三角形
- 题目：题号、题干片段、真题关键词
- 方法：待定系数法、数形结合、分类讨论
- 错因：审题漏条件、计算符号错、不会设未知数
- 真题：近 10 年中考、某地区某年份真题
- 学习路径：我想补一次函数、我想练几何证明
- AI 问法：不会这题怎么办、怎么找题眼

搜索结果不应只是列表。成熟形态应该返回一个小型补法建议：

- 你当前最该补的节点
- 2 道基础题
- 1 个典型题组
- 1 道中考真题
- 相关错因
- 可直接问 AI 的提问词
- 是否加入今天任务

树脉搜索与提问斜杠的边界：

- 树脉搜索：学生知道自己想找什么，主动检索。
- 提问斜杠：学生卡住了，不知道该怎么问，系统递出问法。

二者都应进入 SKE 的反馈闭环。搜索点击、结果停留、加入任务、后续做题正确率，都可以反过来优化知识树路径和 Skill 权重。

后台应新增：

**提示词 / Skill 管理**

基础能力：

- 新增提示词
- 修改提示词

- 今日展示最多的 Skill
- 点击率最高的 Skill
- 有帮助率最高的 Skill
- 负反馈最高的 Skill
- 某知识点缺少 Skill 的题目
- 某题型效果最好的前 5 条提示
- AI 回答质量异常记录

## 九、技术实施建议

### 9.1 第一阶段最小闭环

第一版不要追求复杂训练，先完成：

1. 数据库新增 `prompt_skills` 表
2. 数据库新增 `prompt_skill_events` 表
3. 后端增加推荐 API
4. 题目页展示 3 条“你可以这样问”
5. 学生点击后调用 AI
6. 记录 impression / click / ai_used / helpful / not_helpful
7. 后台可以查看基础统计

只要有记录，系统就开始具备进化能力。

### 9.2 建议数据表

```sql
CREATE TABLE prompt_skills (
  id SERIAL PRIMARY KEY,
  skill_key VARCHAR(80) UNIQUE NOT NULL,
  name VARCHAR(120) NOT NULL,
  type VARCHAR(40) NOT NULL,
  audience VARCHAR(40) NOT NULL,
  subject VARCHAR(40) DEFAULT 'math',
  stage VARCHAR(40) DEFAULT 'middle',
  grade VARCHAR(20),
  topic_code VARCHAR(80),
  question_type VARCHAR(80),
  scene VARCHAR(80),
  goal VARCHAR(80),
  difficulty VARCHAR(30),
  tone VARCHAR(50),
  content TEXT NOT NULL,
  model_role VARCHAR(40) DEFAULT 'user_suggestion',
  weight NUMERIC(5, 3) DEFAULT 0.700,
  version VARCHAR(30) DEFAULT '1.0.0',
  status VARCHAR(20) DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE prompt_skill_events (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE SET NULL,
  prompt_skill_id INTEGER REFERENCES prompt_skills(id) ON DELETE CASCADE,
  question_id VARCHAR(80),
  topic_code VARCHAR(80),
  scene VARCHAR(80),
  event_type VARCHAR(40) NOT NULL,
  result VARCHAR(40),
  meta JSONB DEFAULT '{}',
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 9.3 建议 API

```text
GET  /api/skills/recommend?questionId=xxx&topicCode=xxx&scene=xxx

GET  /api/admin/skills
```

## 十、模型路由与 AI 点数

### 10.1 不建议定位为 AI 中转站

树脉不应把前台做成模型超市。学生不需要关心 DeepSeek、GPT、Claude、Gemini 哪个更强。

更好的定位：

**树脉 AI 引擎：底层模型可切换，前台只呈现更好的学习服务。**

### 10.2 前台能力档位

建议前台显示：

- 标准教练：日常讲题、基础提示、普通错因
- 深度教练：压轴题、多轮追问、复杂诊断
- 冲刺教练：考前整理、专题突破
- 极速答疑：简单题快速回答

后台可对应不同模型和成本。

### 10.3 模型接入原则

- DeepSeek 可作为第一主力模型，成本低、中文能力强
- 高阶模型用于压轴题、复杂追问、答案质检
- 不建议依赖不稳定中转站 Key
- 如支持用户自带 Key，只建议给教师、家长、学校管理员，不建议给普通学生
- 用户 Key 必须后端加密保存，前端不可见，可随时删除
- 发送给第三方模型的数据要做隐私最小化

### 10.4 AI 点数

收费不应叫“买答案次数”，建议叫：

**AI 深度服务额度**

可消耗额度的能力：

- 深度错因诊断
- 压轴题多轮讲解
- 图片识题 / OCR
- 试卷整体分析
- 个性化计划生成
- 家长周报
- 教师班级报告
- 复杂答案质检

基础学习权长期免费，高成本 AI 服务按实际消耗透明计费。

## 十一、商业模式总结

树脉的商业结构建议：

### 11.1 永久免费基础层

- 知识树
- 基础题
- 部分真题
- 每日任务
- 基础错题修复
- 基础 AI 提示

### 11.2 透明共建层

- 成本公开
- 用户自愿共建
- 低价支持服务器和基础 AI 调用
- 共建用户获得更稳定额度和优先体验

### 11.3 高级服务层

- 深度 AI 诊断
- 考前冲刺计划
- 个性化纸质训练包
- 家长报告
- 教师班级管理
- 学校 / 机构部署

### 11.4 核心原则

不要向学生收“学习机会税”。收费应来自更高成本、更高组织复杂度、更深度的服务。

## 十二、质量标准

未来判断树脉内容质量，不只看题库数量，还要看每道题是否完成“教学化”。

优质题目应包含：

- 题干
- 答案
- 标准解析
- 常见错因
- 知识点
- 方法标签
- 分层提示
- 追问引导
- 变式题策略
- AI 调用模板
- 反馈数据

树脉要从“题库质量”升级为“教学智能质量”。

## 十三、阶段路线

### SKE-1 文档与数据结构

- 完成本文档
- 写任务交接卡
- 新增数据库表设计
- 明确 Skill 类型与字段

### SKE-2 最小提示词仓库

- 内置 30-50 条高质量前台提示
- 覆盖基础题、真题、错题、知识点场景
- 接入现有 AI 提示词体系

### SKE-3 题目页引导气泡

- PageDetail / CoachPanel 展示 3 条推荐提示
- 点击后自动发起 AI 对话
- 记录展示和点击

### SKE-3.1 提问斜杠

- 在题目教练场输入框中监听 `/`
- 弹出 3-5 条动态提问词
- 首版采用下拉式，不采用卷帘式或瀑布式
- 点击后调用对应 Skill，并记录 impression / click / ai_used
- 后续根据学生基础、错因和帮助反馈动态调整排序

### SKE-3.2 树脉搜索

- 在总首页和学科主页保留克制的搜索入口
- 支持搜索知识点、题目、方法、错因、真题和 AI 问法
- 搜索结果按最相关、知识点、题目、方法、错题与薄弱点、真题、AI 建议分组
- 成熟阶段把搜索结果升级为补法建议，而不是普通资料列表

### SKE-4 效果记录

- 记录 impression / click / ai_used / helpful / not_helpful
- 记录订正完成、同类题后续结果
- 为后续权重调整提供数据

### SKE-5 后台管理

- 新增 Skill 管理 tab
- 支持增删改查、启停、复制版本、查看统计

### SKE-6 模型路由与点数

- 基础模型 / 深度模型 / 质检模型分层
- 高成本服务接入 AI 点数
- 前台不做模型超市

### SKE-7 质量评测

- 建立题目 + Skill + 模型的质量评测集
- 检测正确性、清晰度、初中生可懂程度、成本

---

## 十四、最终定位

树脉教学 Skill 引擎的使命：

**把每一次学生卡住的瞬间，转化为下一次更精准的引导。**

树脉未来卖的不是题库，也不是模型调用次数，而是：

**经过真实学习数据验证的智能教学能力。**
