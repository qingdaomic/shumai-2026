# V4.50 后台 Skill 管理记录

最后更新：2026-05-09

## 目标

把树脉教学 Skill 从“前台推荐能力”推进到“后台可管理能力”，让管理员可以直接查看、筛选、调整权重和启停状态，形成最小运营闭环。

## 本阶段完成

- 新增后台 `Skill 管理` Tab
- 支持按 `skill_key / 名称 / 内容` 搜索
- 支持按 `type / status / scene / subject` 筛选
- 支持查看 Skill 总数、启用数、停用数、前台可用数
- 支持逐条修改 `weight` 与 `status`
- 后台列表复用 `prompt_skills` 表，不新增数据结构

## 本阶段修改文件

- `src/App.jsx`
- `server/api/admin.js`
- `tasks.md`
- `文档/树脉系统建设记录.md`

## 设计判断

后台 Skill 页面先做“可读、可调、可回收”三件事，不先做复杂编辑器，不先做批量导入，也不把它做成臃肿的配置中心。

这是为了保持树脉后台的节奏：

- 前台先有推荐
- 后台再有管理
- 数据先跑起来
- 运营能力后补齐

## 验证预期

完成后应能：

- 在管理端看到 Skill 列表
- 搜到某个 `skill_key`
- 调高或调低权重
- 把不合适的 Skill 暂时停用

## 边界

- 不改数据库结构
- 不部署
- 不 SSH
- 不碰 `server/schema.sql`
- 不碰 `deploy/`
- 不碰 `.env*`
- 不碰 TTS / ASR
