# V4.61 SKE/PET 学伴动作效果回流记录

最后更新：2026-05-09

## 目标

让学伴动作不只记录“被点击”，还要记录它是否真的触发了 AI 使用，以及学生觉得有没有帮助。

## 本阶段完成

- `recordPetSkillActionEvent()` 支持传入事件类型
- 学伴动作点击继续记录 `click`
- 学伴动作打开 AI 浮窗并成功回答后记录 `ai_used`
- AI 浮窗对学伴来源回答显示反馈：
  - 有帮助
  - 没帮助
- 反馈继续写入 `/api/skills/event`
- metadata 保留：
  - `entry: learning_pet`
  - `action_id`
  - `label`
  - `prompt`
  - `topic_name`
  - `entry_point`

## 设计判断

本阶段仍不新增后端接口和数据库字段。学伴动作沿用 SKE 已有事件类型：`click`、`ai_used`、`helpful`、`not_helpful`。

这样后台 Skill 质量评分可以自然吸收学伴动作效果，不需要另起一套统计系统。

## 边界

- 不新增后端接口
- 不改数据库
- 不执行 SQL
- 不部署
- 不 SSH
- 不 PM2 / Nginx
- 不触碰服务器和禁止文件

## 后续

- 后台 Skill 来源筛选
- 学伴动作根据 `helpful / not_helpful` 做默认排序
- 高效学伴动作可沉淀为首页或题目页的默认建议
