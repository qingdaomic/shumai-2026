## 首次验证发现的问题

首次请求 3002 后台 Skill 管理时发现：

- `permission denied for table prompt_skills`

原因：

- V4.90 migration 由 postgres 用户执行
- 新表 owner / 权限没有自动授予应用连接用户 `shumai`
- 应用后端使用 `shumai` 连接数据库，因此无法访问新建表

影响：

- `prompt_skills` 即使已经建表并导入 41 条 seed，如果不补 GRANT，release 后端仍无法使用后台 Skill 管理
- `admin_operation_logs` 也需要确认应用用户具备写读权限
