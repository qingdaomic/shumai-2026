# V4.52 SKE-7 Skill 质量评测记录

最后更新：2026-05-09

## 目标

让后台 Skill 管理页不只显示配置，还能看到学生行为反馈形成的最小效果分，开始判断哪些提示词值得保留、哪些需要降权或观察。

## 本阶段完成

- 后台 `GET /api/admin/skills` 聚合最近 30 天 `prompt_skill_events`
- 每条 Skill 返回：
  - `impressions`
  - `clicks`
  - `ai_used`
  - `helpful`
  - `not_helpful`
  - `quality_score`
- 后台 Skill 管理页新增质量列
- 后台顶部新增 30 日展示、点击、有帮助、没帮助概览

## 质量分口径

当前为最小可解释分：

```text
quality_score =
  (clicks * 0.5 + ai_used * 1.2 + helpful * 3 - not_helpful * 4)
  / max(impressions, 1)
```

这个分数不是最终教学质量评分，只是第一版运营观察口径。

## 设计判断

先让后台能看见行为反馈，而不是一开始就自动改权重。自动降权需要更稳的样本量和误伤保护，适合后续在 SKE-7.1 做。

## 边界

- 不新增数据库表
- 不执行 SQL
- 不部署
- 不 SSH
- 不 PM2 / Nginx
- 不真实自动降权
- 不触碰 `server/schema.sql`
