## 执行过程

- 复核 `deploy/nginx-shumai.conf`
- 对照 V4.94-V4.98 的线上事实，确认当前正式发布口径：
  - 前端 root：`/opt/shumai/dist`
  - 后端进程：`shumai-api-v4.94`
  - 后端 cwd：`/opt/shumai-releases/v4.94-20260509-190052/repo/server`

发现并修正：

- `DEPLOY.md` 仍出现把 `/opt/shumai` 当作默认 clone 目录的写法
- `DEPLOY.md` 仍示例直接使用仓库内 `server/ecosystem.config.cjs` 启动 `shumai-api-v4.94`，但该文件当前仍指向旧 `/opt/shumai/server`
- `deploy/deploy-backend.sh` 默认猜测 `/opt/shumai-releases/current/repo`，但线上并未固化这个 current 软链
- `deploy/deploy.sh` 使用 `scp` 上传前端，不能删除旧 hash 静态资源
- `.env` 示例曾带有具体连接串 / key 形态，已收敛为键名清单，避免文档携带敏感值格式
