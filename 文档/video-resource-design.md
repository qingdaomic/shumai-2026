# 中考数学系统视频资源管理设计文档

> 版本：V1.0 | 对应阶段：V1.5 开始实现，V2 完整运营

---

## 一、设计目标

视频资源功能不是"往知识点里写死一个 B 站链接"，而是一套完整的**平台级视频资源管理能力**，需要同时支持：

- 内容团队采集和维护视频资源
- 管理端审核和关联知识点
- 学员端按知识点查看推荐视频
- 后台持续检测链接有效性
- 行为数据反哺内容运营决策

---

## 二、核心概念定义

| 概念 | 说明 |
|------|------|
| 视频资源（Video） | 一条外部视频的元数据，包括 URL、标题、时长、UP主等 |
| 主推荐视频 | 每个知识点最多一个，学员端默认展示 |
| 备选视频 | 每个知识点可有多个，供学员自选 |
| 候选池 | 尚未关联知识点或待审核的视频，暂不对学员展示 |
| 质量分 | 管理员审核时对视频打的 1-100 综合分 |
| 适用层次 | basic（基础入门）/ intermediate（提高）/ advanced（冲刺压轴） |

---

## 三、数据模型（与 database-design.md 对应）

核心表：

- `videos` — 视频资源主表
- `topic_video_relations` — 知识点-视频关联表
- `video_click_logs` — 视频点击行为日志

关键字段回顾：

**videos 表核心字段**：

| 字段 | 说明 |
|------|------|
| bvid | B站视频唯一标识，防止重复录入 |
| status | pending / approved / rejected / offline |
| quality_score | 1-100，审核评分 |
| suitable_for | basic / intermediate / advanced |
| is_available | 链接当前是否可访问（定时检测更新） |
| last_checked_at | 最近一次链接检测时间 |

**topic_video_relations 表核心字段**：

| 字段 | 说明 |
|------|------|
| is_primary | 是否为主推荐视频（每个知识点只能有一个） |
| priority | 展示排序，越小越靠前 |
| match_score | 视频与知识点的匹配度 1-100 |
| status | active（展示）/ hidden（隐藏但不删除） |

## 四、视频状态流转

```
新录入
  │
  ▼
pending（待审核）
  │
  ├── 审核通过 ──► approved（已发布）
  │                    │
  │                    ├── 链接失效 ──► offline（已下线）
  │                    │                    │
  │                    │                    └── 恢复 ──► approved
  │                    │
  │                    └── 手动下线 ──► offline
  │
  └── 审核拒绝 ──► rejected（已拒绝）
                       │
                       └── 重新提交 ──► pending
```

**规则**：

- `pending` → `approved`：审核通过
- `pending` → `rejected`：审核拒绝（需填写原因）
- `approved` → `offline`：链接检测失效 或 手动下线
- `offline` → `approved`：链接恢复后手动重新激活
- `rejected` → `pending`：修改后重新提交审核

**学员端只能看到** `status = approved` 且 `is_available = 1` 的视频。

## 五、知识点-视频关联规则

### 5.1 主推荐视频约束

- 每个知识点最多只能有 **1 个** `is_primary = 1` 的视频
- 设置新主视频时，原主视频自动降为备选（`is_primary = 0`）
- 主推荐视频必须是 `status = approved` 且 `is_available = 1`

### 5.2 备选视频约束

- 每个知识点备选视频数量建议不超过 **5 个**
- 按 `priority` 升序排列，数值越小越靠前
- 备选视频可以是不同适用层次（basic / intermediate / advanced）

### 5.3 无视频时的处理

- 若某知识点无任何关联视频，学员端不展示视频区域
- 若主视频下线，自动从备选中按 priority 顺序提升第一个展示
- 后台每日检测，发现主视频失效时通知管理员

## 六、视频采集与录入流程

### 6.1 当前推荐方式：人工辅助录入

针对每个知识点，建议按以下流程操作：

**第一步：确定搜索词**

每个知识点预先定义 2-3 个标准搜索词，例如：

| 知识点 | 搜索词 |
|--------|--------|
| rational（有理数） | 有理数 初中数学 讲解、七年级有理数运算、有理数中考专项 |
| linear_fn（一次函数） | 一次函数 初中 讲解、一次函数中考题型、一次函数图像规律 |
| quad_fn（二次函数） | 二次函数 顶点式、二次函数压轴 中考、二次函数最值 |

**第二步：筛选候选视频**

在 B 站搜索后，按以下标准初筛：

- 标题明确对应知识点
- 时长 5-20 分钟（过短可能内容不完整，过长可能不聚焦）
- UP 主近期活跃，专注数学教育内容
- 播放量不低于 5000（基础质量参考）
- 无明显错误（可快速扫几个知识点关键步骤）

**第三步：录入管理端候选池**

填写：视频 URL / 标题 / UP 主 / 时长 / 适用层次 / 备注

**第四步：管理端审核**

审核员根据质量标准打分，通过后关联知识点。

---

### 6.2 后续阶段：半自动化采集

V2 阶段可引入辅助工具：

- 批量搜索词匹配
- 自动抓取视频基本信息
- 基于标题关键词自动建议知识点关联
- 生成候选清单供人工确认

**注意**：自动化只做"候选推荐"，不做"自动发布"。所有视频必须经过人工审核才能对学员可见。

## 七、视频质量审核标准

审核员对视频打 1-100 分，参考以下维度：

| 维度 | 权重 | 评分说明 |
|------|------|---------|
| 知识点匹配度 | 30% | 视频内容是否准确对应本知识点，无偏题或跑题 |
| 学段匹配度 | 20% | 是否符合初中/中考体系，不超纲不低于目标学段 |
| 讲解清晰度 | 20% | 结构清楚，步骤规范，逻辑完整 |
| 内容正确性 | 20% | 无数学错误，公式正确，解题过程严谨 |
| 资源稳定性 | 10% | UP 主账号稳定，视频长期有效可能性高 |

**发布阈值**：综合评分 ≥ 70 分方可通过审核。

**拒绝场景举例**：

- 视频内容与知识点不符
- 存在明显数学错误
- 内容超纲（如用高中知识解初中题）
- 视频质量极差（音质/画质无法正常观看）
- 视频已被 B 站删除或设为私密

## 八、学员端展示规范

### 8.1 知识点详情页视频区域

展示位置：概念定义下方，关联题目上方。

**有主推荐视频时**：

```
┌─────────────────────────────────────────┐
│  视频讲解                               │
│  ┌───────────────────────────────────┐  │
│  │  🎬 有理数基础讲解                │  │
│  │  某某老师 · 11分20秒 · 基础入门   │  │
│  │                   [ 去 B站 观看 ] │  │
│  └───────────────────────────────────┘  │
│  备选视频：[ 有理数进阶 ] [ 中考专项 ]  │
└─────────────────────────────────────────┘
```

**无视频时**：

不显示视频区域，不展示空状态占位。

### 8.2 智能诊断结果页视频推荐

当诊断出薄弱知识点后，在推荐练习旁附带视频入口：

```
薄弱知识点：有理数
推荐练习  ▶ 5道基础题
视频讲解  🎬 有理数基础讲解  [观看]
```

### 8.3 错题本页视频关联

错题详情中展示对应知识点的视频入口，方便学员"看完视频再复习这道题"。

## 九、链接有效性检测机制

### 9.1 检测时机

- **定时任务**：每天凌晨 2 点检测全部 `status = approved` 的视频
- **即时检测**：管理员手动触发单条或批量检测

### 9.2 检测方式

对每个视频 URL 发起 HTTP HEAD 请求：

- 返回 200：`is_available = 1`，更新 `last_checked_at`
- 返回 404 / 403 / 其他错误：`is_available = 0`
- 连续 3 次失效：自动将 `status` 改为 `offline`，触发通知

### 9.3 失效处理

- 自动下线，学员端不再展示
- 管理端视频列表标红提示
- 若该视频为某知识点主视频，自动降级为备选中第一个有效视频
- 后台发送站内提醒给管理员

---

## 十、视频行为数据采集

每次学员点击"去 B 站观看"时，前端调用 `POST /video-clicks`，记录：

- `video_id`：哪个视频
- `topic_code`：从哪个知识点触发
- `source_page`：从哪个页面触发（topic / diagnosis / wrong_book）
- `user_id`：学员 ID（未登录为 null）

**后续可统计**：

- 各知识点视频点击量排名
- 点击率最高的视频（可作为质量参考）
- 诊断后视频观看转化率
- 错题关联视频使用率

## 十一、管理端视频中心模块功能清单

| 功能 | 说明 |
|------|------|
| 视频候选池 | 展示所有 pending 状态视频，供审核 |
| 已发布视频 | approved 状态，可查看点击量、下线操作 |
| 已拒绝视频 | rejected 状态，可查看拒绝原因 |
| 已下线视频 | offline 状态，可重新激活 |
| 审核操作 | 通过/拒绝，填写质量分和拒绝原因 |
| 知识点关联 | 将视频关联到一个或多个知识点 |
| 主视频设置 | 为知识点指定主推荐视频 |
| 链接检测 | 手动触发检测，查看失效报告 |
| 批量录入 | 支持 CSV / JSON 格式批量导入视频信息 |
| 数据统计 | 各视频点击量、各知识点视频覆盖情况 |

## 十二、视频资源覆盖目标

当前系统共有 **24 个核心知识点**（`TOPICS` 数组），按阶段规划视频覆盖：

| 阶段 | 目标覆盖 | 说明 |
|------|---------|------|
| V1.5 启动 | 重点 8 个知识点 | 优先覆盖高频考点：有理数、一次函数、二次函数、三角形、圆、概率、统计、方程 |
| V2 正式 | 全部 24 个知识点 | 每个知识点至少 1 个主推荐视频 |
| V2.5 精细 | 全部知识点 2-3 个 | 覆盖 basic / intermediate / advanced 各层次 |

## 十三、实施顺序建议

1. **现在（V1 测试版）**：学员端预留视频区域 UI，无视频时不展示，不写死链接
2. **V1.5 轻后端**：建立 `videos` 和 `topic_video_relations` 表，管理端可手动录入和关联
3. **V1.5 上线前**：完成 8 个重点知识点的视频采集、审核、关联
4. **V2**：完整 24 个知识点覆盖 + 链接检测 + 行为统计 + 诊断/错题视频联动
