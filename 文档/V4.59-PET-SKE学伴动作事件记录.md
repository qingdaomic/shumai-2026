# V4.59 PET/SKE 学伴动作事件记录

最后更新：2026-05-09

## 目标

把学伴的三个教学动作接入 SKE 事件体系，让它们后续可以被后台统计和质量评估。

## 本阶段完成

- 给三个学伴动作绑定正式 Skill：
  - `给我一个突破口` -> `math_common_hint_first_step_001`
  - `判断错因` -> `math_common_error_diagnose_001`
  - `来一道更简单的` -> `math_common_simplify_same_type_001`
- 新增 `recordPetSkillActionEvent()`
- 学伴动作触发时记录 `/api/skills/event`
- 事件类型使用 `click`
- metadata 标记：
  - `entry: learning_pet`
  - `action_id`
  - `label`
  - `prompt`
  - `topic_name`

## 设计判断

本阶段不新增事件类型，继续复用 SKE 已支持的 `click`，把“来自学伴”写入 metadata。这样不会改数据库，不会改后端事件白名单，也能被现有 Skill 质量统计继续吸收。

## 边界

- 不新增后端接口
- 不改数据库
- 不执行 SQL
- 不部署
- 不 SSH
- 不 PM2 / Nginx
- 不触碰服务器和禁止文件

## 后续

- 后台 Skill 质量页可增加来源筛选：斜杠提问 / 搜索 / 学伴
- 学伴动作后续可记录 `ai_used` 与 `helpful`
- 低效学伴动作可降权，高效动作可沉淀为默认推荐
