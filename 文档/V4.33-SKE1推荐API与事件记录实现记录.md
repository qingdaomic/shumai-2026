## 2. 新增文件

### `server/services/prompt-skills.js`

职责：

- 标准化推荐参数。
- 优先从 `prompt_skills` 数据库表读取 Skill。
- 数据库不可用或表未迁移时，读取 `server/seeds/prompt_skills_seed.json`。
- 按 `topic_code`、`method_code`、`scene`、`question_type`、`question_id`、`error_type`、`student_state` 和 `weight` 排序。
- 记录 `prompt_skill_events`。
- 事件写入失败时返回 `stored:false`，不让接口崩溃。

关键导出：

- `recommendPromptSkills(input)`
- `recordPromptSkillEvent(input)`
- `normalizeRecommendInput(input)`
- `rankSkills(skills, params)`

### `server/api/skills.js`

职责：

- 提供 Express Router。
- `POST /recommend`
- `POST /event`
- 做基础参数校验。
- 将 service 结果整理成前端可读格式。

注意：

- 本阶段没有挂载到 `server/index.js`。
- 后续挂载建议：

```js
import skillsRouter from './api/skills.js';
app.use('/api/skills', skillsRouter);
```

## 3. 推荐 API 设计

后续挂载后路径：

```text
POST /api/skills/recommend
```

请求体：

```json
{
  "topic_code": "quad_fn",
  "question_id": "fg01_0",
  "question_type": "function",
  "stage": "middle",
  "error_type": "method_choice",
  "student_state": "stuck",
  "scene": "question_detail",
  "limit": 3
}
```

返回：

```json
{
  "skills": [
    {
      "skill_key": "math_topic_quadratic_function_breakthrough_001",
      "name": "二次函数破局",
      "type": "hint",
      "content": "二次函数题先别展开算。请提示我第一步要定什么...",
      "topic_code": "quad_fn",
      "method_code": "structure_first",
      "scene": "stuck_before_solution",
      "weight": 0.94
    }
  ],
  "source": "db",
  "reason": "已从数据库按知识点 quad_fn / 题型 function / 场景 question_detail 推荐"
}
```

如果数据库不可用或未迁移：

```json
{
  "skills": [],
  "source": "seed",
  "reason": "数据库不可用或尚未迁移，已使用种子 Skill 降级推荐..."
}
```

实际 fallback 会返回 seed 中排序后的 3 条 Skill。

## 4. 事件 API 设计

后续挂载后路径：

```text
POST /api/skills/event
```

请求体：

```json
{
  "skill_key": "math_common_hint_first_step_001",
  "event_type": "click",
  "topic_code": "quad_fn",
  "question_id": "fg01_0",
  "question_type": "function",
  "scene": "question_detail",
  "metadata": {
    "source": "question_detail"
  }
}
```

支持事件：

- `impression`
- `click`

返回：

```json
{
  "ok": true,
  "stored": true,
  "reason": "事件已写入 prompt_skill_events"
}
```

数据库不可用或表未迁移时：

```json
{
  "ok": true,
  "stored": false,
  "reason": "数据库不可用或表尚未迁移，事件降级为 no-op..."
}
```

## 5. fallback 策略

推荐接口：

- 先查数据库。
- 数据库表不存在、数据库不可用、连接失败时，自动读取 `server/seeds/prompt_skills_seed.json`。
- seed 推荐仍按相同排序函数执行。

事件接口：

- 先根据 `skill_id` 或 `skill_key` 找数据库 Skill。
- 找不到或数据库不可用时，不报死，返回 `stored:false`。
- 这样前端未来可以先展示推荐，不被迁移状态卡住。
