## 上线前必须复核的差异

### 1. health 服务名差异

当前线上旧后端：

- 3001 health name：`树脉后端`

GitHub release 后端：

- health name：`数脉后端`

影响：

- 功能层面不阻塞
- 但品牌命名不一致
- `https://shumai.cc/api/health` 是可访问接口，建议不要带着这个差异正式切换

建议：

- 正式切换前先做一个极小代码校准，把 release 后端 health 与启动日志统一回 `树脉后端`
- 校准后重新做一次 3002 health + 后台 Skill API 快速复验

### 2. 正式 PM2 命名策略

当前线上：

- root PM2：`shumai-api:online`
- 监听：3001
- 当前运行目录：`/opt/shumai/server`

建议正式切换时不要立即删除旧进程：

- 先停止旧 `shumai-api`
- 新 release 以 `shumai-api-v4.94` 监听 3001
- 验证通过后暂不 `pm2 save`
- 稳定观察后再决定是否替换为长期进程名或执行 `pm2 save`

理由：

- 旧 `shumai-api` 保持 stopped 状态，便于快速 rollback
- 新进程名带版本，便于识别
- 不保存 PM2 可以降低错误状态固化风险

### 3. 前端发布时间

正式切换顺序建议：

1. 先准备并构建前端 `dist`
2. 先切后端 3001 到 release
3. 验证 `/api/health` 与后台 API
4. 再 rsync 前端 `dist` 到 `/var/www/shumai`
5. 验证公网页面与后台入口

理由：

- 新前端依赖后台 Skill 管理 / 操作日志 API
- 如果先发前端而后端仍是旧代码，后台新入口可能失败

## V4.94 正式切换操作卡

以下为下一阶段可执行操作草案。执行前必须再次确认模型建议为 GPT-5.5 high，并明确“允许服务器切换”。

### 1. 新建正式 release

```bash
sudo mkdir -p /opt/shumai-releases
sudo chown ubuntu:ubuntu /opt/shumai-releases
RELEASE=/opt/shumai-releases/v4.94-$(date +%Y%m%d-%H%M%S)
git clone https://github.com/qingdaomic/shumai-2026.git "$RELEASE/repo"
cd "$RELEASE/repo"
git rev-parse HEAD
git status --short
```

验收：

- HEAD 等于 GitHub 最新 `origin/main`
- `git status --short` 为空

### 2. 构建与语法检查

```bash
cd "$RELEASE/repo"
npm ci
npm run build
cd "$RELEASE/repo/server"
npm ci --omit=dev
node --check index.js
```

验收：

- 前端 `dist/` 生成
- server 依赖安装成功
- `node --check index.js` 通过

### 3. 切换前快照

```bash
SNAP=/home/ubuntu/shumai-backups/v4.94-cutover-$(date +%Y%m%d-%H%M%S)
mkdir -p "$SNAP"
sudo rsync -a /var/www/shumai/ "$SNAP/var-www-shumai-before/"
sudo pm2 jlist > "$SNAP/pm2-before.json"
curl -fsS http://127.0.0.1:3001/api/health > "$SNAP/health-3001-before.json"
```

验收：

- 前端静态目录快照存在
- PM2 列表已留档
- 旧 3001 health 已留档

### 4. 3002 最终旁路确认

```bash
cd "$RELEASE/repo/server"
sudo env PORT=3002 NODE_ENV=production DOTENV_CONFIG_PATH=/opt/shumai/server/.env \
  pm2 start index.js --name shumai-api-v4.94-precheck --no-autorestart --time
curl -fsS http://127.0.0.1:3002/api/health
```

还需要服务器内生成短期管理员 token，验证：

- `GET /api/admin/skills`
- `POST /api/skills/recommend`

验收后：

```bash
sudo pm2 stop shumai-api-v4.94-precheck
sudo pm2 delete shumai-api-v4.94-precheck
```

### 5. 后端 3001 切换

```bash
sudo pm2 stop shumai-api
cd "$RELEASE/repo/server"
sudo env PORT=3001 NODE_ENV=production DOTENV_CONFIG_PATH=/opt/shumai/server/.env \
  pm2 start index.js --name shumai-api-v4.94 --no-autorestart --time
curl -fsS http://127.0.0.1:3001/api/health
curl -fsS https://shumai.cc/api/health
```

验收：

- 本机 3001 health 正常
- 公网 `/api/health` 正常
- PM2 中旧 `shumai-api` 为 stopped，新 `shumai-api-v4.94` 为 online
- 暂不 `pm2 save`

### 6. 后台 API 验收

服务器内生成短期管理员 token，验证：

- `GET /api/admin/skills?limit=20`
- `GET /api/admin/operation-logs?limit=10`
- `POST /api/skills/recommend`

验收：

- `summary.total` 约为 41
- operation logs 返回 `stored:true`
- SKE recommend 返回 `source:db`

### 7. 前端发布

```bash
sudo rsync -a --delete "$RELEASE/repo/dist/" /var/www/shumai/
```

验收：

- `https://shumai.cc/` 可访问
- `https://shumai.cc/?view=admin` 可访问
- 登录后后台 Skill 管理页面不 500
- 后台操作历史入口可打开

### 8. 观察期

观察 10-15 分钟：

- `curl -fsS https://shumai.cc/api/health`
- root PM2 中新后端持续 online
- Nginx 不需要 reload
- 后台 API 无 500

观察通过后，另开 V4.95 决定是否：

- 删除旧 stopped `shumai-api`
- 将新进程转为长期命名
- 执行 `pm2 save`

## 回滚卡

### 后端回滚

如果新 3001 异常：

```bash
sudo pm2 stop shumai-api-v4.94
sudo pm2 delete shumai-api-v4.94
sudo pm2 restart shumai-api
curl -fsS http://127.0.0.1:3001/api/health
curl -fsS https://shumai.cc/api/health
```

预期：

- 旧 `shumai-api` 恢复 online
- 3001 health 恢复旧响应

### 前端回滚

如果前端异常：

```bash
sudo rsync -a --delete "$SNAP/var-www-shumai-before/" /var/www/shumai/
```

预期：

- 静态文件恢复到切换前版本
- 不需要 reload Nginx

### 数据库回滚原则

本轮数据库变更均为 additive，且已通过 V4.88/V4.90/V4.91 验证。

正式切换时不建议做破坏性数据库回滚。若数据库出现异常，先回滚应用后端与前端，再单独评估数据库处理。

## Go / No-Go 清单

可以进入正式切换的条件：

- GitHub main 已包含命名校准或明确接受 `数脉后端` health 名称
- 最新 release 目录构建通过
- 3002 最终旁路验证通过
- DB 基线符合：
  - `prompt_skills=41`
  - `prompt_skill_events=0` 或仅有已知可解释事件
  - `admin_operation_logs=0` 或仅有已知可解释日志
- `shumai` 应用用户具备三张后台相关表和 sequence 权限
- `/var/www/shumai` 切换前快照已完成
- root PM2 切换前列表已留档

停止上线的条件：

- 3002 health 失败
- `GET /api/admin/skills` 失败
- `POST /api/skills/recommend` 不是 `source:db`
- 新 3001 health 失败
- 公网 `/api/health` 失败
- 前端构建失败
- 任意命令需要临时改 Nginx、改线上 DB schema、改 `/opt/shumai`
