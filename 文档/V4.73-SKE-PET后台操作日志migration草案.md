# V4.73 SKE/PET 后台操作日志 migration 草案

日期：2026-05-09

## 目标

V4.72 已经明确后台操作日志需要独立于学生行为事件表。V4.73 按这个边界新增最小 migration 草案，只定义 `admin_operation_logs` 表和索引，不执行 SQL，不连接数据库。

## 本阶段完成

- 新增 `server/migrations/20260509_admin_operation_logs.sql`
- 表名：`admin_operation_logs`
- 用途：记录管理员对 Skill 等后台对象的人工操作
- 覆盖字段：
  - 管理员：`admin_user_id`、`admin_role`
  - 操作：`action`
  - 目标：`target_type`、`target_id`、`target_key`、`target_name`
  - 前后摘要：`before_value`、`after_value`
  - 来源与原因：`source`、`reason`
  - 排查链路：`request_id`
  - 时间：`created_at`
- 新增索引：
  - 按时间倒序
  - 按目标对象
  - 按操作类型
  - 按管理员
  - 按 `target_key`

## 设计取舍

- `before_value` / `after_value` 使用 `jsonb`，只保存摘要，不强制保存完整大文本。
- `admin_user_id` 使用 `ON DELETE SET NULL`，避免管理员账号变化导致日志丢失。
- `action` 和 `target_type` 增加非空白约束，避免写入无意义日志。
- `target_key` 单独建部分索引，方便按 Skill key 回看历史操作。
- 不复用 `prompt_skill_events`，继续保持学生行为事件和管理员审计日志分离。

## 边界

- 不执行 SQL
- 不连接本地或线上数据库
- 不修改 `server/schema.sql`
- 不新增 API
- 不改前端
- 不部署
- 不服务器操作
- 不 SSH
- 不 PM2 / Nginx / PostgreSQL
- 不触碰 TTS / ASR / study-plan / deploy / public animations / dist-singlefile / local-notes / `.env*`
