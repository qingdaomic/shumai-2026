# V4.60 SKE/PET 后台 Skill 来源统计记录

最后更新：2026-05-09

## 目标

让后台 Skill 管理页能看出 Skill 事件来自哪里：斜杠提问、AI 回答、学伴动作、搜索等来源，为后续评估学伴动作是否有效做准备。

## 本阶段完成

- `GET /api/admin/skills` 新增来源聚合：
  - `slash_prompt`
  - `skill_answer`
  - `learning_pet`
  - `search`
  - `unknown`
- 每条 Skill 返回来源计数：
  - `source_slash_prompt`
  - `source_skill_answer`
  - `source_learning_pet`
  - `source_search`
  - `source_unknown`
- 后台 Skill 管理页新增来源汇总 chip
- Skill 表格新增“来源”列
- 学伴动作点击后，后台可看到 `learning_pet` 来源事件

## 设计判断

本阶段不改数据库结构，直接复用 `prompt_skill_events.meta.entry`。这样可以继续沿用已有事件表与质量统计，避免为了“来源”单独加字段或新表。

## 边界

- 不新增数据库表
- 不执行 SQL
- 不部署
- 不 SSH
- 不 PM2 / Nginx
- 不触碰 TTS / ASR / study-plan / deploy / schema 等禁止或历史脏改文件

## 后续

- 学伴动作触发 AI 后补记 `ai_used`
- 学伴回答反馈可补 `helpful` / `not_helpful`
- 后台后续可增加来源筛选，而不只是展示来源统计
