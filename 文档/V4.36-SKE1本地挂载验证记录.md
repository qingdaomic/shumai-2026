## 2. 新增验证文件

新增：

- `server/scripts/verify-skills-api.js`

脚本行为：

- 创建本地临时 Express app。
- 挂载 `skillsRouter` 到 `/api/skills`。
- 使用本地短期测试 JWT 通过现有 `authMiddleware`。
- 调用 `POST /api/skills/recommend`。
- 调用 `POST /api/skills/event`，分别验证 `impression` 和 `click`。
- 验证完成后自动关闭临时 server。

说明：

- 脚本只监听本机 `127.0.0.1` 随机端口。
- 不连接线上服务器。
- 不执行 SQL。
- 不修改数据库。
- 不写入真实密钥、账号、token。

## 3. 推荐接口验证

验证请求：

```json
{
  "topic_code": "quad_fn",
  "question_id": "fg01_0",
  "question_type": "function",
  "stage": "middle",
  "error_type": "method_choice",
  "student_state": "stuck",
  "scene": "question_detail",
  "limit": 3
}
```

验证结果：

- HTTP 状态：200
- `source`: `seed`
- 返回 Skill 数量：3
- 返回 Skill：
  - `math_topic_quadratic_function_breakthrough_001`
  - `math_topic_function_image_reading_001`
  - `math_common_hint_first_step_001`

结论：

本地 PostgreSQL 未连接或 SKE 表尚未迁移时，`/api/skills/recommend` 可以正常走 seed fallback，并返回 3 条推荐 Skill。

## 4. 事件接口验证

已验证事件：

- `impression`
- `click`

验证结果：

- `impression`：HTTP 200，`ok:true`，`stored:false`
- `click`：HTTP 200，`ok:true`，`stored:false`

返回语义：

数据库不可用或 `prompt_skill_events` 尚未迁移时，事件接口降级为 no-op，不让前端流程崩溃。

结论：

SKE-1 的最小事件记录接口具备上线前的安全降级特性。后续只要完成 migration 并挂载路由，就可以把 `stored:false` 逐步切换为真实写入。

## 5. 为什么本阶段不修改 `server/index.js`

V4.28 / V4.31 已确认 `server/index.js` 当前不是一个干净的 SKE 挂载点，而是同时混有：

- TTS / ASR 路由挂载
- study-plan 路由挂载
- 动画 cron 启动
- 其他后端实验能力

如果本阶段直接提交 `server/index.js`，容易把不该上线的后端能力一起带入 GitHub 或未来服务器同步。

因此本阶段只新增验证脚本，不修改、不提交 `server/index.js`。
