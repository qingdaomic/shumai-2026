# V4.63 SKE/PET 学伴动作反馈排序记录

日期：2026-05-09

## 目标

把学伴动作从固定顺序展示，升级为会吸收反馈的推荐顺序。

前面 V4.59 到 V4.62 已经完成：

- 学伴动作点击进入 Skill 事件
- AI 使用记录 `ai_used`
- 学生反馈记录 `helpful` / `not_helpful`
- 后台可按来源统计和筛选

V4.63 的重点是让学生端先用这些信号做一个最小闭环：反馈好的动作排在前面，反馈差的动作进入观察。

## 本阶段完成

- `bumpSlashLocalStat()` 在本地统计变化后派发 `shumai-skill-local-stats` 事件
- 学伴面板监听本地 Skill 统计变化并刷新动作排序
- 新增学伴动作评分：
  - 点击小幅加分
  - 触发 AI 使用中等加分
  - 有帮助强加分
  - 没帮助强扣分
- 学伴动作按钮展示状态标签：
  - 优先建议
  - 反馈较好
  - 已被使用
  - 有人点击
  - 需观察
  - 默认建议

## 设计边界

- 不新增接口
- 不改数据库
- 不执行 SQL
- 不部署
- 不服务器操作
- 不 SSH
- 不 PM2 / Nginx
- 不触碰 TTS / ASR / study-plan / deploy / schema / public animations 等禁止或历史脏改文件

## 说明

这一版先使用浏览器本地的 Skill 统计，优点是安全、轻量、可立即验证。后续如果要做跨设备一致排序，可以再让学生端读取后端按用户聚合后的 Skill 效果数据。

- 后台增加“低效学伴动作”观察清单
- 后续可把来源筛选、质量分、学生反馈组合成推荐调整建议
