# 中考数学系统前端设计文档

## 1. 文档目的

本文件用于指导前端测试版以及后续正式学员端、管理端前端的演进设计。当前重点是：在保证现有功能可运行的前提下，让前端结构逐步具备接入后端的能力。

## 2. 当前前端基础

现有项目以 `shumai-v7-1.jsx` 为核心，属于单文件 React 应用，包含：

- 页面导航
- 知识点数据
- 题库数据
- 智能诊断逻辑
- 错题本逻辑
- 本地持久化逻辑
- AI 调用逻辑

该结构适合快速原型，但不适合长期维护。后续前端设计要以“逐步拆分而不是一次性推倒”为原则。

## 3. 前端目标

### 3.1 测试版目标

- 快速上线学员可用版本
- 页面稳定可访问
- 核心学习路径完整
- 保留现有知识点和题库价值
- 预留未来接口接入点

### 3.2 正式版目标

- 支持学员端与管理端分离
- 接入后端 API
- 支持用户登录与数据同步
- 支持视频资源、支付、行为统计等扩展模块

## 4. 推荐前端分层

### 4.1 页面层

负责：

- 路由与页面布局
- 页面级状态组织
- 页面组件装配

建议页面类型：

- 首页
- 知识点详情页
- 基础习题页
- 题组训练页
- 压轴突破页
- 智能诊断页
- 错题本页
- 个人中心页
- 登录/注册页
- 管理端页面集合

### 4.2 组件层

负责：

- 知识卡片
- 题目卡片
- 筛选组件
- 统计组件
- 视频推荐卡片
- 诊断结果组件
- 弹窗、按钮、标签、空状态等通用组件

### 4.3 服务层

这是当前最需要补的层。

建议抽象：

- `topicService`
- `questionService`
- `diagnosisService`
- `wrongBookService`
- `userService`
- `videoService`
- `analyticsService`
- `billingService`

当前阶段这些 service 可以先读本地数据；后续再无缝替换为请求 API。

### 4.4 存储层

需要对以下数据统一封装：

- 本地缓存
- localStorage
- 会话状态
- 登录 token
- 草稿数据

建议通过 `storageService` 或统一 hooks 管理，而不是在组件中分散读写。

## 5. 学员端页面结构建议

### 5.1 首页

展示：

- 学习入口
- 今日推荐
- 最近学习进度
- 诊断提醒
- 错题数量
- 视频推荐入口

### 5.2 知识点详情页

展示：

- 知识点名称
- 概念定义
- 公式与结论
- 易错点
- 对应方法
- 关联基础题
- 关联题组
- 推荐视频入口

### 5.3 基础习题页

展示：

- 题目列表
- 按知识点筛选
- 难度筛选
- 作答状态
- 收藏/错题操作

### 5.4 智能诊断页

展示：

- 诊断入口
- 诊断结果
- 薄弱知识点
- 推荐练习
- 推荐视频

### 5.5 错题本页

展示：

- 错题列表
- 按知识点/时间/状态筛选
- 再练一次
- 加入复习计划
- 查看对应知识点和视频

### 5.6 个人中心

测试版可先预留：

- 基础资料
- 学习统计
- 收藏数
- 错题数
- 诊断次数
- 会员/套餐入口占位

## 6. 管理端前端建议

虽然当前优先做学员端，但应预留未来管理端结构，管理端建议单独站点或独立路由模块。

管理端页面可包括：

- 登录页
- 仪表盘
- 学员管理
- 知识点管理
- 题库管理
- 视频资源管理
- 审核页
- 收费管理
- 行为统计页
- 系统设置页

## 7. 状态管理建议

### 7.1 当前阶段

可使用 React 本地状态 + context + 本地存储封装。

### 7.2 后续阶段

当页面和模块拆分后，可引入统一状态管理方案，但应避免过早复杂化。

关键是先统一状态边界：

- 用户态
- 学习态
- 筛选态
- 题目作答态
- 诊断结果态
- 错题态
- 视频资源态

## 8. 数据模型约束

前端应尽量只依赖统一模型，而非依赖源文件中任意字段。建议逐步标准化：

- `Topic`
- `Question`
- `QuestionGroup`
- `DiagnosisResult`
- `WrongQuestionRecord`
- `VideoResource`
- `UserProfile`

## 9. 交互与体验要求

### 9.1 核心要求

- 页面不可出现黑屏或点击无响应
- 筛选与切换需清晰
- 移动端和桌面端均可使用
- 保持学习路径连续性

### 9.2 视频功能预留

知识点详情页和诊断结果页应预留视频区域：

- 主推荐视频
- 备选视频
- 无视频时的空状态
- 观看跳转按钮

## 10. 重构策略建议

不建议一次性把单文件拆完，建议按以下顺序重构：

- 先抽 service 层
- 再抽通用组件
- 再抽页面组件
- 再把数据从 UI 文件中逐步分离
- 最后再接后端 API

## 11. 前端验收标准

- 核心功能稳定可用
- 页面数据来源具备服务层封装
- 未来可平滑接入后端
- 管理端设计边界清晰
- 视频、注册、收费等新功能已预留扩展点
