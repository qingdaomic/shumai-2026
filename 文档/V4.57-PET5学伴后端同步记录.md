# V4.57 PET-5 学伴后端同步记录

最后更新：2026-05-09

## 目标

让树脉学伴从纯本地状态，走向登录用户可跨设备同步的成长状态；同时把 XP 奖励事件放到服务端去重，避免同一有效学习行为被反复刷分。

## 本阶段完成

- 新增 `server/api/pet.js`
- 新增独立 migration 草案 `server/migrations/20260509_pet_sync.sql`
- 新增后端接口：
  - `GET /api/pet`：读取当前用户学伴状态
  - `PUT /api/pet`：同步形态、状态、XP、最近奖励
  - `POST /api/pet/events`：记录 XP 奖励事件，并用 `event_key` 做去重
- 前端 `LearningPet` 接入登录态：
  - 未登录继续使用本地 `localStorage`
  - 已登录会尝试读取云端学伴状态
  - XP 奖励会带 `rewardKey` 写入 `/api/pet/events`
  - 切换小猫 / 小狗、切换状态会同步到 `/api/pet`

## 数据结构草案

### `pet_states`

- `user_id`
- `species`
- `mode`
- `xp`
- `level`
- `last_reward`
- `created_at`
- `updated_at`

### `pet_events`

- `user_id`
- `event_type`
- `event_key`
- `xp_delta`
- `label`
- `meta`
- `created_at`

`event_key` 用于防刷，同一用户同一个奖励键只能写入一次。

## 设计判断

本阶段没有改 `server/schema.sql`，避免混入历史数据库脏改；只新增独立 migration 草案。真正上线前需要由总控确认数据库变更窗口，再执行 migration。

前端保持降级能力：如果后端未部署、表未创建或接口不可用，学伴仍然按本地保存工作，不影响学生当前体验。

## 边界

- 不执行 SQL
- 不部署
- 不 SSH
- 不 PM2 / Nginx
- 不修改 `server/schema.sql`
- 不触碰 TTS / ASR / SVG / WeChat / animation / deploy 相关文件
