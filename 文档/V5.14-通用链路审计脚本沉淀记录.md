# V5.14 通用链路审计脚本沉淀记录

日期：2026-05-09

## 目标
把“知识点-基础题-题组-压轴-真题”的上下串联检查沉淀成一个可复用的本地脚本，后续小学数学、高中数学和其他链路审计都能直接沿用。

## 模型建议
本阶段以本地文档与脚本沉淀为主，建议模型：
- 纯文档 / 本地审查：GPT-5.4-Mini
- 如果以后扩展到服务器或线上数据核验，再切 GPT-5.5 high

## 执行前状态
- 当前仓库已有大量历史脏改，本阶段只补一个新的通用审计脚本和对应记录，不清理、不 reset
- 当前主链路数据已有基础题、题组、压轴、真题和知识图谱，但覆盖并不全
- V1.5 / V1.6 继续暂停，不纳入本阶段

## 执行过程
- 新增 `scripts/audit-link-chain.mjs`
- 新增 `npm run audit:links`
- 脚本读取知识点、基础题、题组、压轴、真题和 `src/App.jsx` 页面入口文案，自动输出覆盖率、缺口和关键词命中
- 结构上做成了配置驱动，后续只要换路径和导出名，就能复用到小学/高中或别的审计场景

## 验证结果
- 脚本已能直接作为本地审计器使用
- 计划用它持续检查：
  - 知识点是否有上下链路
  - 基础题是否有向上提升
  - 题组 / 压轴是否挂在对应节点上
  - 真题是否保留知识和方法入口
  - 页面上是否真的露出上下串联按钮
