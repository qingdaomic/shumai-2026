# 数脉 ShuMai 中考数学智能学习系统

## 项目概述

面向初中生的数学 AI 私人教练，整合知识图谱、分模块学习、1334 道题库、错题本、智能诊断、AI 讲解和微信互动。

**核心能力**：
- 194 个知识点技能树，覆盖初中三年完整数学体系
- 1334 道精选数学题（624 真题 + 500 基础 + 137 题组 + 58 压轴 + 15 诊断）
- 每道题/每个知识点一键 AI 讲解（DeepSeek）
- 微信 ClawBot 集成：学生在微信里和 AI 学伴对话、接收学习推送
- 双学习模式：正向递进（知识→题目）+ 逆向归因（试卷→知识点→补漏）
- PWA 支持：安卓手机/Pad "添加到主屏幕"像 App 体验

**技术栈**：
- 前端：React 18 + Vite 5，部署 Netlify
- 后端：Node.js + Express + PostgreSQL（搭建中）
- 微信：ClawBot（腾讯官方 iLink 协议）
- AI：DeepSeek API

**在线访问**：https://shumai-2026.netlify.app

## 核心特性

- **知识图谱学习**
  - 以 24 个核心知识节点为图谱主干
  - 支持节点点击跳转到知识详情
  - 支持已掌握状态可视化

- **四模块学习法**
  - 基础知识
  - 基础习题
  - 题组训练
  - 压轴题组

- **23 种解题方法体系**
  - 将方法与知识点建立映射
  - 支持方法分类浏览与关联知识点跳转

- **中考真题训练**
  - 内置多年真题数据集 `EXAM_QS`
  - 支持按知识点、年份、城市、难度筛选
  - 支持错题收录与错题本联动

- **错题本**
  - 自动按知识点分组
  - 展示错因追溯和补差路径
  - 支持语音讲解文稿生成与朗读

- **智能诊断**
  - 分析未掌握高频考点
  - 统计错题分布
  - 沿知识依赖树追溯根源
  - 自动生成推荐学习路径
  - 调用 AI 输出学情诊断报告

- **中考预测**
  - 基于近年真题统计生成命题预测
  - 输出必考点、高概率点、复习优先级与方法建议

- **每日晨读卡**
  - 基于 `MORNING_DATA` 构造知识点晨读内容
  - 支持卡片式记忆和结果记录

- **本地持久化**
  - 使用 `localStorage` 保存掌握状态、错题、基础错题、AI 设置等

## 页面结构

应用的页面由 `view` 状态驱动，导航配置位于 `NAV`。

### 页面列表

- **首页 `home`**
  - 入口仪表盘
  - 展示掌握进度、知识域统计、错题本统计、快捷入口等

- **晨读卡 `morning`**
  - 每日记忆与快速回顾页面

- **知识图谱 `graph`**
  - 可视化展示知识点节点与关联关系

- **四模块 `modules`**
  - 系统化分层学习主页面

- **知识详情 `detail`**
  - 单个知识点详情页
  - 展示讲解、前置知识、典型题与练习入口

- **方法体系 `methods`**
  - 23 种解题方法分类浏览

- **真题训练 `practice`**
  - 真题库浏览、筛选、解析和错题收录

- **错题本 `wrong`**
  - 按知识点聚合错题
  - 展示根因分析和语音讲解

- **智能诊断 `diag`**
  - 学情诊断、风险预警、学习路径与 AI 诊断报告

- **中考预测 `predict`**
  - 基于历年数据的命题趋势预测

每个方法对象通常包含：

常见字段包括：

- `name`
- `domain`

- `examYears`
- `totalScore`

### 4. 真题题库 `EXAM_QS`

系统题库主数据源，字段通常包括：

- `id`
- `yr`
- `no`
- `city`
- `type`
- `topic`
- `score`
- `diff`
- `subTopics`
- `methods`
- `content`
- `answer`
- `sol`
- `error`
- `upLink`
- `downLink`

### 5. 晨读数据 `MORNING_DATA`

为每个知识点提供：

- `slogan`

## 核心逻辑说明

### 路由与导航

应用没有使用外部路由库，而是通过根组件里的状态切换页面：

- `view`
- `detailId`

入口函数：

- `navigate(v, tid = null)`

逻辑：

- 如果传入 `tid`，进入 `detail` 页并记录知识点 ID
- 否则切换普通页面视图

此外还监听了全局事件：

- `shumai-nav`

用于子组件触发跨页面跳转。

### 本地状态持久化

根组件通过 `localStorage` 管理以下状态：

- `mastered`
- `wrongSet`
- `basicWrongSet`
- `aiModel`
- `dsKey`
- `dbKey`

存储键：

- `shumai_v7`

保存方式：

- 初次加载时从本地恢复
- 状态变化时自动写回

### 知识根因追溯

核心函数：

- `traceRootCause(topicId, masteredSet)`

逻辑：

- 基于 `DEPENDENCIES` 进行 BFS 向下搜索
- 找出当前知识点依赖链中未掌握的前置知识
- 返回：
  - `unmastered`
  - `roots`

该函数被以下模块复用：

- 错题本
- 智能诊断
- 学习路径推荐

### AI 调用

兼容调用入口：

- `callAI(system, user, history = [])`
- `callClaude = callAI`

能力包括：

- 智能诊断报告生成
- 中考预测报告生成
- 错题语音讲解文稿生成

注意：

- 需要联网
- 需要配置有效 API Key
- 未配置时会返回提示文本，不会自动完成云端分析

### 语音讲解

相关函数：

- `fetchMSTTS(text)`
- `speakText(text, onStart, onEnd)`
- `fallbackSpeak(text, onStart, onEnd)`
- `stopSpeak()`
- `genVoiceScript(q, topicName)`

机制：

- 优先使用微软 Edge 朗读接口合成音频
- 失败时回退到浏览器原生 `speechSynthesis`

## 已知实现特点与限制

- **单文件实现规模大**
  - 所有设计、数据、逻辑、页面都在一个 `jsx` 文件中
  - 便于单文件交付，但不利于长期维护

- **内置数据量大**
  - 真题、知识点、方法体系、晨读数据都写在源码里
  - 易于离线分发，但会使文件体积变大

- **AI 能力依赖网络**
  - 离线单文件版 UI 可运行
  - 但 AI 生成能力仍依赖外部 API

- **存在少量数据重复键警告**
  - 某些题目对象里有重复的 `answer` 或 `sol` 字段
  - 构建可通过，但后者会覆盖前者
  - 建议后续做数据清洗

## 近期优化建议（V1 测试版阶段）

- **完善错误边界**
  - 给各页面增加 React Error Boundary，避免局部异常导致整页黑屏

- **修复数据质量问题**
  - 清理 `EXAM_QS` 中题目对象的重复字段（`answer` / `sol`）
  - 建立数据校验脚本

- **引入 service 层**
  - 将数据访问从组件中抽出，通过统一 service 函数访问
  - 便于后续无缝替换为后端 API

- **预留视频区域 UI**
  - 知识点详情页预留视频展示区域
  - 无视频时不展示，不写死任何链接

- **前端部署**
  - 参考 `文档/deployment.md` 中的 V1 静态部署方案

## 维护建议

### 单文件阶段（当前）

如果继续以单文件维护 `shumai-v7-1.jsx`，建议遵循：

- 按区域分块维护，每个大块用注释标记边界
- 保持数据结构一致性，新增字段先确认已有数据模型
- 所有新增页面统一走 `NAV + view` 机制
- 所有新增持久化状态统一收口到 `loadStorage / saveStorage`
- 所有 AI 新能力统一复用 `callAI`

### 平台化阶段（V1.5 起）

- 前端代码逐步迁移为多文件工程结构
- 数据从源码迁移至数据库（参考 `文档/database-design.md`）
- 所有接口按 `文档/api-design.md` 定义实现
- 用户体系、权限控制参考 `文档/permissions-design.md`
